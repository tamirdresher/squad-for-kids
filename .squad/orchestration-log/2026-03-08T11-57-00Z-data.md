## Summary

Data identified and fixed critical bug in ralph-watch.ps1 causing squad-monitor v2 dashboard to display empty "Ralph Recent Rounds" panel despite the watch script running continuously.

**Root Cause:** Missing variable definitions for `$maxLogBytes` and `$maxLogEntries` in the Invoke-LogRotation function, causing PowerShell to silently fail during log write operations.

**Resolution:** Added variable initialization at script head:
- `$maxLogBytes = 1MB`
- `$maxLogEntries = 500`

## Notes for Team

**PowerShell Lessons Learned:**
- Undefined variables in conditionals cause silent failures (no error thrown)
- Recommend `Set-StrictMode -Version Latest` during development
- Always test observability outputs (logs, heartbeat files) before deploying watch loops
- Log rotation thresholds should be configurable for different scale tiers

**Next Steps:** Consider exposing log rotation config as environment variables for future watch script deployments.
