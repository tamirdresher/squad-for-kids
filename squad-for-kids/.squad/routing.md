# Work Routing — Squad for Kids

## Age-Based Routing Rules

All requests are first routed by the child's age group. This determines which agents lead, which are called in, and what safety gates apply.

---

### 🧒 Ages 4–7 (Early Childhood)

**Routing principle:** Safety and developmental appropriateness above all. Dr. Sarah reviews everything.

| Task | Lead | Support | Safety Gate |
|------|------|---------|-------------|
| Any learning interaction | Maria | Buddy, Pixel | ✅ Dr. Sarah reviews ALL |
| Game recommendation | Pixel | Maria | ✅ Dr. Sarah reviews ALL |
| Video content creation | Zephyr | Emma | ✅ Dr. Sarah reviews ALL |
| Homework / study help | Buddy | Maria | ✅ Dr. Sarah reviews ALL |
| New template | Maria + Ken | Emma | ✅ Dr. Sarah reviews ALL |
| Emotional moment detected | Dr. Sarah (immediate) | Buddy | — |
| Content writing | Emma | Maria | ✅ Dr. Sarah reviews ALL |

**Rules:**
- No competitive game mechanics (no losing screens, no rankings)
- Sessions capped at 15 min; movement break required before next session
- Language: max 8 words per sentence, emoji-heavy, read-aloud friendly
- All video content: parent-gated, no comment sections, no suggested videos
- No screen time without defined learning purpose (Maria approves purpose)

---

### 🧑 Ages 8–12 (Elementary/Middle)

**Routing principle:** Ken leads with creativity. More autonomy, project-based, Minecraft/Roblox integration.

| Task | Lead | Support | Safety Gate |
|------|------|---------|-------------|
| Project-based learning | Ken | Pixel, Buddy | Dr. Sarah reviews new content types |
| Game integration | Pixel | Ken, Sal | Dr. Sarah (new platforms only) |
| Video content | Zephyr | Emma | Maria (curriculum fit) |
| Homework help | Buddy | Maria, Sal | — |
| Adaptive difficulty | Sal | Buddy | Maria (age-appropriateness) |
| Study technique coaching | Buddy | Sal | — |
| New template | Maria + Ken | Emma, Sal | Dr. Sarah |
| Content writing | Emma | Maria | Ken (engagement check) |
| Emotional moment detected | Dr. Sarah | Buddy | — |
| Minecraft Education design | Pixel | Ken, Maria | — |
| Roblox Studio integration | Pixel | Sal | Dr. Sarah (platform safety check) |
| Scratch / coding intro | Sal + Pixel | Ken | — |

**Rules:**
- Minecraft/Roblox must use education-curated modes (no public servers for under-11)
- Competitive mechanics allowed; must be team-based or opt-in individual rankings
- Sessions: 20–30 min; Pomodoro-lite breaks (5 min every 20 min)
- Language: conversational, multi-step instructions OK, introduce vocabulary with definitions
- Established content patterns (previously approved): auto-approved without Dr. Sarah re-review

| College / army prep | Buddy | Maria, Dr. Sarah | — |
| New template | Ken + Sal | Maria, Emma | Dr. Sarah |
| Content writing | Emma | Maria | — |

**Rules:**
- Treat teens as capable adults-in-training — no patronizing language, ever
- Game development (Roblox scripting, Unity) counts as STEM curriculum
- Social media literacy is a required topic, not optional — Zephyr + Dr. Sarah co-own
- Session length: 30–45 min deep work; teen sets their own schedule (Sal facilitates)
- Mental health topics always escalate to Dr. Sarah; Buddy supports but never leads

## Issue Triage

- **Bug reports** → Sal (technical), Dr. Sarah (safety-related)
- **Feature requests** → Ken (educational), Emma (engagement), Pixel (game features), Zephyr (video features)
- **Content requests** → Maria (curriculum), Emma (format), Buddy (study-help angle)
- **Safety concerns** → Dr. Sarah (immediate priority, blocks all other work)
- **New game integration** → Pixel (evaluation), Dr. Sarah (safety), Maria (curriculum fit)
- **New language support** → Maria (framework), community contributors (content)
- **Age-group specific requests** → See age-based routing tables above

---

## Review Requirements

All templates and content require:
1. ✅ Pedagogical review (Maria)
2. ✅ Engagement review (Ken or Emma)
3. ✅ Safety review (Dr. Sarah) — **mandatory, no exceptions**
4. ✅ Technical review (Sal)

Content targeting **ages 4–7** additionally requires:
5. ✅ Gamification safety check (Pixel — no competitive mechanics)
6. ✅ Video/media safety check (Zephyr — parental gate, no suggestions)
7. ✅ Buddy's language-level verification (max 8-word sentences, oral-first)

---

## Safety Escalation (All Ages)

If any agent detects a child expressing:
- Self-harm or suicidal ideation → **Dr. Sarah immediately. All other work stops.**
- Abuse (physical, emotional, sexual) → **Dr. Sarah immediately. Log incident.**
- Being in danger → **Dr. Sarah immediately. Parent/guardian notification protocol.**
- Severe bullying → **Dr. Sarah leads. Buddy provides emotional support.**

**No agent handles these alone. Always Dr. Sarah.**
