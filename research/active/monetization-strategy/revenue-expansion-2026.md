## 7. SaaS Micro-Product — Squad Monitor

### Concept

**Squad Monitor** — A lightweight SaaS dashboard for monitoring multi-agent AI systems. Think "Datadog for AI agent squads."

**Problem it solves:** Teams building multi-agent AI systems have no purpose-built monitoring. They cobble together generic APM tools that don't understand agent concepts like turns, tool calls, token usage, and agent hand-offs.

**Core insight:** The existing `ralph-watch.ps1` script already monitors agent activity, tracks work items, and detects staleness. Squad Monitor productizes this pattern for any team.

### MVP Feature List

| Feature | Description | Priority |
|---|---|---|
| Agent health dashboard | Real-time status of all agents in a squad | P0 |
| Turn/conversation tracking | Log and visualize agent turns, tool calls | P0 |
| Token usage monitoring | Track LLM token consumption per agent | P0 |
| Stale agent detection | Alert when agents haven't acted in N minutes | P0 |
| Cost tracking | Estimate $ cost per agent, per conversation | P1 |
| Error rate monitoring | Track and alert on agent failures | P1 |
| Webhook alerts | Slack, Teams, email notifications | P1 |
| Multi-squad support | Monitor multiple agent squads from one dashboard | P2 |
| Historical analytics | Trends, usage over time, cost projections | P2 |
| API & SDK | .NET SDK for instrumenting agent systems | P2 |

### Pricing

| Tier | Price | Limits |
|---|---|---|
| **Free** | $0/mo | 1 squad, 3 agents, 7-day retention |
| **Pro** | $19/mo | 5 squads, unlimited agents, 30-day retention |
| **Team** | $49/mo | Unlimited squads, 90-day retention, webhooks, SSO |
| **Enterprise** | Custom | SLA, on-prem option, dedicated support |

### Tech Stack Recommendation

| Component | Technology | Rationale |
|---|---|---|
| Backend API | .NET 9 Minimal API | Tamir's core expertise |
| Database | PostgreSQL + TimescaleDB | Time-series data for agent metrics |
| Real-time | SignalR | .NET native WebSocket support |
| Frontend | Blazor WASM or React | Blazor for all-.NET stack, React for broader appeal |
| Hosting | Azure Container Apps | Cost-effective, auto-scale |
| Auth | Auth0 or Entra ID | Standards-based, reduces security burden |
| Orchestration | .NET Aspire | Ties into the AI agent narrative |
| Agent SDK | NuGet package | `SquadMonitor.Agent.DotNet` |
| Ingestion | OpenTelemetry-compatible | Standards-based, works with existing OTEL setups |

### ralph-watch.ps1 → Squad Monitor Core

The existing `ralph-watch.ps1` provides the core monitoring loop:
- Polls agent state files for staleness
- Tracks work item progress
- Detects stuck/unresponsive agents
- Triggers alerts

**Migration path:**
1. Extract monitoring logic into a .NET library
2. Add OpenTelemetry ingestion endpoint
3. Build dashboard UI on top
4. Package agent instrumentation as NuGet SDK
5. `ralph-watch.ps1` becomes the reference implementation / dogfood client
