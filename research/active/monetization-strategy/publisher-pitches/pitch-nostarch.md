**THE STAR TREK ANGLE (THIS IS THE FUN PART)**
Each AI agent has a crew persona that shapes its behavior:
- **Picard** — The Lead. Makes architecture decisions, resolves conflicts between agents, delegates work. ("Make it so.")
- **Data** — The Code Expert. Writes C#, Go, clean code. Precise, thorough, slightly too literal.
- **Worf** — Security & Cloud. Reviews every PR for vulnerabilities, manages Azure infrastructure. Suspicious of everything, as security people should be.
- **Seven of Nine** — Research & Docs. Deep research, documentation, analysis. Processes information with Borg-like efficiency.
- **B'Elanna** — Infrastructure. Kubernetes, Helm charts, ArgoCD. Gets angry at broken YAML (relatable).
- **Troi** — Communications. Blog writing, voice matching, storytelling. The empathic one.
- **Ralph** — The Watch Monitor. Runs a 5-minute continuous loop watching for new work. Not a Star Trek character — he's the intern who never sleeps.

This isn't decoration. The personas are a design pattern. They create natural boundaries for agent responsibilities, make the system memorable and teachable, and — frankly — make the book a lot more fun to read than "Chapter 3: Agent Decomposition Patterns."
