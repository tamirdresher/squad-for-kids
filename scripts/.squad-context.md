## What This Area Does

The scripts area is the **automation and tooling layer** for the tamresearch1 system. It contains
PowerShell scripts, Python utilities, and Node.js helpers that drive squad operations, monitoring,
email/calendar automation, podcast generation, devbox management, and deployment support.

Key sub-areas:
- **Squad operations** — `squad-watch.ps1`, `ralph-watch*.ps1`, `daily-squad-report.ps1`, `validate-squads-config.ps1`
- **GitHub / auth helpers** — `gh-account-helpers.ps1`, `gh-account-wrappers.ps1`, `gh-emu.cmd`, `gh-personal.cmd`
- **DevBox management** — `devbox-*.ps1` scripts for provisioning, keep-alive, SSH setup, and startup
- **Email & notifications** — `send-squad-email*.ps1`, `ralph-email-monitor.ps1`, `birthday-*.ps1`, `check-personal-email.ps1`
- **Podcast generation** — `generate-podcast-script.py`, `voice-clone-podcast.py`, `f5tts-podcast-runner.py`, `deliver-podcast.ps1`
- **Monitoring & metrics** — `model-monitor.ps1`, `code-churn-rate.ps1`, `rate-limit-*.ps1`, `teams-board-dashboard.ps1`
- **Setup & onboarding** — `setup-*.ps1` scripts for credentials, secrets, webhooks, and squad member onboarding
- **Tech/news scanning** — `tech-news-scanner.js`, `arxiv-scanner.js`, `eyal-links-monitor.js`
- **FedRamp baseline** (`scripts/fedramp-baseline/`) — compliance baseline scripts
- **Drift detection** (`scripts/drift-detection/`) — infrastructure drift detection
- **Smoke tests** (`scripts/smoke-tests/`) — system smoke test suite
- **Measurement** (`scripts/measurement/`) — metric collection scripts
- **Squad email** (`scripts/squad-email/`) — email pipeline scripts

## Routing Hints

- **New squad automation scripts** → Data implements; B'Elanna reviews if infra-impacting
- **DevBox / cloud resource scripts** → B'Elanna primary review
- **Auth / secrets / credential setup scripts** (`setup-*.ps1`) → Worf mandatory review
- **GitHub CLI wrappers** (`gh-*.ps1`, `gh-*.cmd`) → Data; Worf reviews any auth changes

- **Email / notification scripts** → Data; Ralph for scheduling integration

- **Tech news / ArXiv scanners** → Data; Seven for research integration
- **FedRamp baseline scripts** → Data + Worf mandatory review

## Constraints

- All scripts must be idempotent — safe to re-run without side effects
- Never hard-code secrets, tokens, or passwords — use environment variables, Key Vault, or Bitwarden
- PowerShell is the default language; use Python only where PowerShell is insufficient (e.g., ML/audio)
- Node.js scripts are for web scraping/API calls only; heavy logic goes in PowerShell or Python
- `gh auth switch --user tamirdresher_microsoft` must precede all `gh` CLI calls that target the org account
- Scripts touching production systems require B'Elanna review before merge
- Credential setup scripts (`setup-secrets.ps1`, `setup-squad-credentials.ps1`) require Worf sign-off

## Conventions

- PowerShell scripts use `.ps1` extension; shell scripts use `.sh` only for cross-platform necessity
- Script names use kebab-case (`daily-squad-report.ps1`, not `DailySquadReport.ps1`)
- Each script directory with multiple related files includes a `README` or `*_README.md`
- Setup scripts follow `setup-{target}.ps1` naming convention
- DevBox scripts follow `devbox-{action}.ps1` naming convention
- Python scripts use type hints and virtual environments where applicable
- `__pycache__/` is gitignored
