## איך מערכת הFork עובדת

```
┌──────────────────────────────┐
│  tamirdresher/squad-for-kids │  ← ריפו מקורי (תכנית לימודים, תכונות, תבניות)
│  (upstream)                  │
└──────────┬───────────────────┘
           │ Fork
           ▼
┌──────────────────────────────┐
│  שם-המשתמש/squad-for-kids   │  ← העותק האישי של הילד
│  (הFork שלכם)                │
│                              │
│  📄 student-profile.json     │  ← זהות והתקדמות הילד
│  📁 .squad/reports/          │  ← דוחות שבועיים ללמידה
│  📄 .squad/teaching-plan.md  │  ← התקדמות בתכנית הלימודים
│  📁 .squad/decisions.md      │  ← יומן החלטות הצוות
└──────────────────────────────┘
```

**מושגי מפתח:**
- 🍴 **Fork** = עותק אישי של הריפו. שינויים שלכם לא משפיעים על המקור.
- 🔄 **Upstream** = הריפו המקורי `tamirdresher/squad-for-kids`. אפשר למשוך ממנו עדכונים.
- 📝 **Commits** = כל פגישת למידה יוצרת רשומה בהיסטוריה של הFork שלכם.

## הקמה צעד אחר צעד

### דרישות מקדימות
- **חשבון GitHub** (חינם) — [הירשמו כאן](https://github.com/signup)
- **מחשב נייד או נייח** (לא טלפון/טאבלט)
- **דפדפן** (Chrome, Edge או Firefox)

### 1. עשו Fork לריפו

1. היכנסו ל-[github.com/tamirdresher/squad-for-kids](https://github.com/tamirdresher/squad-for-kids)
2. לחצו על כפתור **"Fork"** (פינה ימנית עליונה, ליד ⭐ Star)
   > 📸 *כפתור הFork נראה כמו חץ מתפצל. הוא בין Watch ל-Star.*
3. בדף "Create a new fork":
   - **Owner:** בחרו את חשבון הGitHub שלכם
   - **Repository name:** השאירו כ-`squad-for-kids`
   - ✅ סמנו "Copy the main branch only"
   - לחצו **"Create fork"**
4. חכו כמה שניות — תועברו לFork החדש!

### 2. ודאו שאתם על הFork שלכם

הסתכלו בפינה השמאלית העליונה של הדף. צריך להיות כתוב:

```
שם-המשתמש-שלכם / squad-for-kids
forked from tamirdresher/squad-for-kids
```

⚠️ **טעות נפוצה:** ודאו שאתם לא על `tamirdresher/squad-for-kids`. אם תיצרו Codespace על הריפו המקורי, ההתקדמות של הילד לא תישמר בחשבון שלכם.

### 3. צרו Codespace

1. בדף **הFork שלכם**, לחצו על הכפתור הירוק **"Code"**
2. עברו ללשונית **"Codespaces"**
3. לחצו **"Create codespace on main"**
4. ⏱️ חכו דקה-שתיים עד שהסביבה נטענת

> 💡 Codespace הוא מחשב ענן שמריץ VS Code בדפדפן שלכם. אין צורך בהתקנה!

### 4. הפעילו את הצוות

1. פתחו Copilot Chat: לחצו על אייקון 💬 בצד, או הקישו `Ctrl+Alt+I`
2. בתחתית חלון הצ'אט, בחרו **"squad"** מתפריט הסוכנים
3. לחצו על **"Autopilot (Preview)"** כדי להפעיל מצב אוטונומי
4. תנו לילד את המקלדת! 🎉

### 5. פגישה ראשונה

הילד כותב ברכה (למשל `היי!`) והצוות עושה את השאר:
- שואל את השם
- שואל את הגיל/הכיתה
- נותן לבחור ערכת נושא (הארי פוטר, גיבורי-על, מיינקראפט...)
- יוצר את צוות הלמידה האישי
- מציע פרויקט ראשון

## סנכרון עדכונים מהריפו המקורי

כשמוסיפים תכונות, תבניות או תכני לימוד חדשים לריפו המקורי, הFork שלכם יכול למשוך את העדכונים:

### אוטומטי (ממשק GitHub)

1. היכנסו לFork שלכם ב-GitHub
2. חפשו את הבאנר: **"This branch is X commits behind tamirdresher:main"**
3. לחצו **"Sync fork"**
4. לחצו **"Update branch"**

> ✅ **בטוח:** הקבצים האישיים של הילד (`student-profile.json`, `.squad/reports/` וכו׳) לא יידרסו — הם לא קיימים בריפו המקורי.

### ידני (שורת פקודה)

אם מעדיפים את הטרמינל (או אם יש merge conflicts):

```bash
# הוסיפו את הremote של המקור (פעם אחת)
git remote add upstream https://github.com/tamirdresher/squad-for-kids.git

# משכו ומזגו עדכונים
git fetch upstream
git merge upstream/main

# דחפו לFork שלכם
git push origin main
```

### באמצעות סקריפט ההקמה

סיפקנו סקריפט PowerShell שמאטמט את ההקמה הראשונית:

```powershell
pwsh setup-parent.ps1
```

הסקריפט:
- ✅ בודק אם הריפו הוא Fork
- 🔗 מגדיר את הremote של `upstream`
- 📄 יוצר תבנית `student-profile.json` (אם לא קיים)
- 👋 מדפיס הודעת ברוכים הבאים

## פרטיות ובטיחות

### מה נשאר פרטי בFork שלכם

| מידע | מיקום | מי יכול לראות |
|------|-------|---------------|
| שם וגיל הילד | `student-profile.json` | רק אתם (הFork פרטי כברירת מחדל) |
| היסטוריית למידה | `.squad/reports/` | רק אתם |
| שיחות צ'אט | Codespace (זמני) | נמחקות כשהCodespace נמחק |
| התקדמות בתכנית לימודים | `.squad/teaching-plan.md` | רק אתם |

### נראות הFork

כברירת מחדל, forks של ריפו ציבורי הם גם ציבוריים. כדי להפוך את הFork של הילד לפרטי:

1. היכנסו לFork → **Settings** → **General**
2. גללו ל-**"Danger Zone"**
3. לחצו **"Change visibility"** → בחרו **"Private"**

> ⚠️ הערה: הפיכת fork לפרטי עשויה לדרוש חשבון GitHub Pro (חינמי לחינוך). לחלופין, אפשר ליצור ריפו פרטי חדש ולהעתיק את הקבצים במקום לעשות fork.

### מינימום מידע

הצוות אוסף רק:
- שם פרטי (להתאמה אישית)
- גיל/כיתה (להתאמת תכנית לימודים)
- עיר (אופציונלי, לזיהוי תכנית לימודים)

**אין שמות משפחה, אין כתובות, אין טלפונים, אין מיילים שנאספים מילדים.**

### מנגנוני בטיחות

- סינון תוכן מותאם גיל לכל קבוצת גיל
- הסוכנים מפנים נושאים לא מתאימים בעדינות
- עזרה בשיעורי בית = הסבר מושגים, לא נתינת תשובות להעתקה
- זיהוי תסכול עם תגובת אמפתיה

## כמה ילדים

### אפשרות א׳: Forks נפרדים (מומלץ)

צרו fork נפרד לכל ילד:

1. **ילד ראשון:** עשו fork רגיל לחשבון שלכם (`שם-משתמש/squad-for-kids`)
2. **ילד שני:** עשו fork שוב, אבל שנו את השם:
   - עשו fork לריפו
   - היכנסו ל-Settings → שנו שם ל-`squad-for-kids-שרה` (או שם הילד)
3. כל ילד מקבל Codespace, פרופיל ומעקב התקדמות משלו

### אפשרות ב׳: ענפים (Branches)

השתמשו בfork אחד עם ענפים שונים לכל ילד:

```bash
# צרו ענף לכל ילד
git checkout -b maya-learning
# ... מאיה משתמשת בענף הזה ...

git checkout -b david-learning
# ... דוד משתמש בענף הזה ...
```

> 💡 **טיפ:** אפשרות א׳ (forks נפרדים) פשוטה יותר ושומרת על הפרדה נקייה. אפשרות ב׳ חוסכת במשאבי Codespace אבל דורשת יותר ידע טכני.

## GitHub Actions ואוטומציה

סקוואד לילדים כולל אוטומציות שרצות על שרתי GitHub — חינם לריפו ציבורי.

### אוטומציות פעילות

| אוטומציה | תפקיד | לוח זמנים | עלות |
|----------|--------|-----------|------|
| `squad-heartbeat.yml` | בודק issues לא מסווגים, מזכיר על PRs ישנים | כל 6 שעות | חינם |
| `squad-triage.yml` | מסווג issues חדשים (באג, תכונה, שיעורי בית, משחק) | בהוספת תגית | חינם |
| `squad-board-sync.yml` | שומר על לוח הפרויקט מסונכרן | באירועים + שבועי | חינם |
| `squad-issue-assign.yml` | מקצה עבודה לחברי הצוות או Copilot | בתגית `squad:` | חינם |

### ניהול אוטומציות

- **כיבוי אוטומציה:** לשונית Actions → לחצו על האוטומציה → `...` → "Disable workflow"
- **הפעלה ידנית:** רוב האוטומציות תומכות ב-`workflow_dispatch` — לחצו "Run workflow"
- **הוספת אוטומציה:** צרו קובץ `.yml` ב-`.github/workflows/`

> 📖 **מדריך מפורט:** [מדריך GitHub Actions ואוטומציה](advanced/github-actions-guide-he.md)

## פתרון בעיות

### "אני לא מוצא/ת את כפתור הFork"

ודאו שאתם מחוברים ל-GitHub. כפתור הFork מופיע בפינה הימנית העליונה של דף הריפו, בין "Watch" ל-"Star".

### "ההתקדמות של הילד נעלמה"

בדקו לאיזה ריפו הCodespace מחובר. אם פתחו Codespace על הריפו **המקורי** במקום על הFork, ההתקדמות נשמרה שם (ועלולה להימחק כשהCodespace נמחק). תמיד ודאו שאתם על הFork שלכם.

### "סנכרון Fork מראה merge conflicts"

זה נדיר אבל יכול לקרות. הגישה הבטוחה:
1. לחצו **"Discard changes"** על קבצי תבנית (בלבד)
2. אם הקונפליקט ב-`student-profile.json` או בדוחות — שמרו את הגרסה שלכם

### "הCodespace איטי או לא נטען"

- נסו לרענן את הדפדפן
- מחקו את הCodespace וצרו חדש (המידע שלכם ב-git, אז זה בטוח)
- בדקו ב-[githubstatus.com](https://www.githubstatus.com) אם יש תקלות שירות

### "סוכן Squad לא מופיע"

1. ודאו שאתם ב-Codespace (לא רק צופים בקבצים ב-GitHub)
2. בדקו ש-Copilot Chat פתוח (אייקון 💬 או `Ctrl+Alt+I`)
3. חפשו בתחתית חלון הצ'אט את תפריט הסוכנים
4. אם "squad" לא ברשימה, נסו לרענן את הCodespace
