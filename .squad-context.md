> Copy this file into any `src/<area>/` directory and adapt for the area.

## Routing Hints

- **Changes to `routing.md`** require Picard approval before merge.
- **New agent charters:** Picard authors, Crusher safety-reviews.
- **Script changes:** Data implements, B'Elanna reviews for infra impact.
- **Documentation changes:** Seven drafts, Picard reviews for accuracy.
- **Security-sensitive files** (credentials, auth logic): Worf mandatory review.
- **Cross-area decisions** stay in root `.squad/decisions/` — never in area decisions/.

## Area Labels

Apply these GitHub labels when work touches this area:

| Label | When |
|-------|------|
| `area:infra` | CI/CD, GitHub Actions, deployment scripts |
| `area:docs` | Documentation, READMEs, guides |
| `area:shared` | Cross-cutting changes affecting multiple areas |

## Conventions

- Decision records go in `.squad/decisions/inbox/{agent}-{brief-slug}.md` (not directly in `decisions/`).
- Agent charters must include `## Identity & Access` section (see #1151).
- All cross-area decisions stay in root `.squad/decisions/` — never in area decisions/.
- Scripts are PowerShell (`.ps1`) unless cross-platform need requires shell (`.sh`).
- `--force` flag on `gh label create` is safe — idempotent re-runs.
