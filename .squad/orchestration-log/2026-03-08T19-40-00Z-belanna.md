## Design Decisions

1. **Microsoft Universal Base:** Pre-built tools, reduced setup time
2. **Two-Layer Setup:** Dockerfile + devcontainer.json for flexibility
3. **Post-Create Automation:** No manual CLI installation required
4. **MCP Config Reuse:** Single source of truth (`.copilot/mcp-config.json`)
5. **Manual Authentication:** User runs `copilot configure` after opening (security)

## Key Characteristics

**Strengths:**
- ✅ Fast setup (2-3 min)
- ✅ Cost effective (GitHub quota vs. Azure compute)
- ✅ Reproducible (config in Git)
- ✅ Team friendly (identical setup for all)
- ✅ Copilot ready (CLI + Squad pre-configured)
- ✅ MCP discoverable (auto-configured)

**Tradeoffs:**
- Stateless by default (stops after 30 min inactivity)
- Container resources < full VM
- Cold start (first open 2-3 min)
- Manual authentication (Phase 2 enhancement)
