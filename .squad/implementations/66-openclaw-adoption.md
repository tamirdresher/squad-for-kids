### 1.2 QMD Extraction Workflow (Weekly)

**Goal:** Automate weekly digest compaction using the QMD framework.

**Reference:** `.squad/templates/qmd-extraction.md`

**Tasks:**

1. **Create QMD Extraction Script** (`.squad/scripts/qmd-extract.ps1`):
   - Input: All daily digests from the past 7 days (`.squad/digests/YYYY-MM-DD.md`)
   - Process: Apply 5-category extraction (KEEP: Decisions, Commitments, Pattern Changes, Blockers, Contacts)
   - Output: One QMD digest (`.squad/digests/archive/YYYY-MM-DD-to-DD.qmd.md`)
   - Report: Signal-to-noise ratio (items kept / items scanned)

   **Key Implementation Details:**
   - Use template categories from `qmd-extraction.md` § "5-Category Extraction — KEEP"
   - Drop categories per § "5-Category Extraction — DROP"
   - Require LLM judgment or regex patterns for KEEP/DROP classification
   - Include quality checklist before finalizing output

2. **GitHub Actions Automation** (`.github/workflows/qmd-weekly.yml`):
   - **Trigger:** Weekly (Sundays 22:00 UTC, configurable)
   - **Job:** Run `qmd-extract.ps1` with current week's dailies
   - **Output:** Generate `.squad/digests/archive/{week}.qmd.md`
   - **Commit:** Auto-commit QMD digest if new items detected
   - **Notification:** Post summary to Squad channel (or log artifact)

3. **Monitoring Metrics:**
   - Items scanned this week
   - Items kept vs. dropped
   - Signal-to-noise ratio (target: ≥30% kept, <70% noise)
   - Processing time

**Acceptance:** 
- Script and workflow created and committed
- First QMD digest generated by end of Week 1
- At least 2 weeks of QMD history by end of implementation

## Phase 2: Dream Routine & Issue-Triager (Week 2)

### 2.1 Dream Routine Scheduling

**Goal:** Automate cross-digest analysis every Monday morning.

**Reference:** `.squad/templates/dream-routine.md`

**Tasks:**

1. **Create Dream Routine Executor** (`.squad/scripts/dream-routine.ps1`):
   - Input: Last 4 QMD digests (default, configurable)
   - Process: 6 analysis tasks (Trending Topics, Recurring Blockers, Incident Clusters, Decision Drift, Skill Promotion, Commitment Tracking)
   - Output: Dream Report (`.squad/digests/dream/dream-YYYY-MM-DD.md`)
   - Gate: Require minimum 2 QMD digests; mark "Low confidence" if <3

   **Implementation Notes:**
   - Use LLM (Claude Sonnet or GPT-4) to perform analysis tasks
   - Provide Dream Routine prompt template from `dream-routine.md`
   - Capture structured output (tables, lists) per "Dream Report Output Format"

2. **GitHub Actions Automation** (`.github/workflows/dream-routine.yml`):
   - **Trigger:** Weekly (Mondays 08:00 UTC, configurable)
   - **Job:** Run `dream-routine.ps1` on last N QMD digests
   - **Output:** Generate `.squad/digests/dream/dream-{date}.md`
   - **Commit:** Auto-commit Dream Report
   - **Notification:** Post findings to #squad-leads or incident channel

3. **Monitoring Metrics:**
   - Analysis confidence level (Low/Medium/High)
   - Number of trending topics detected
   - Recurring blockers identified
   - Incident clusters found
   - Skill promotion candidates flagged

**Acceptance:**
- Script and workflow created
- First Dream Report generated in Week 2
- Minimum 2 QMD digests available for input

### 2.2 Issue-Triager Integration

**Goal:** Integrate Issue-Triager into channel-scan workflow for automatic classification.

**Reference:** `.squad/templates/issue-triager.md`

**Tasks:**

1. **Create Triager Classifier** (`.squad/scripts/issue-triager.ps1`):
   - Input: Per-channel scans (`.squad/digests/{date}-{channel}.md`)
   - Process: Classify each item into one category (Incident, Decision, Question, Coordination)
   - Priority: Apply 3-dimension scoring (blast radius, time sensitivity, reversibility) per template § "Priority Labels"
   - Output: Classified items with priority labels → `.squad/digests/triage/triage-{month}.jsonl`

   **JSONL Schema:**
   ```json
   {
     "id": "TRIAGE-{date}-{seq}",
     "date": "2026-03-11",
     "channel": "dk8s-support",
     "original_item": "{source text}",
     "category": "incident|decision|question|coordination",
     "priority": "P0|P1|P2|P3",
     "blast_radius": "high|medium|low",
     "time_sensitivity": "high|medium|low",
     "reversibility": "high|medium|low",
     "escalation_flag": false,
     "escalation_reason": null,
     "classified_by": "issue-triager-v1",
     "classifier_confidence": 0.95
   }
   ```

2. **P0/P1 Escalation Rules:**
   - **P0:** Incident category + (blast_radius=high AND time_sensitivity=high)
   - **P1:** Incident category, OR Decision blocking other work
   - **Auto-escalate P0** to incident channel / on-call notification

3. **Integration with Channel-Scan:**
   - Modify `.squad/scripts/channel-scan.md` Step 5 to call Issue-Triager after classification
   - Feed classified items to triage JSONL before merging into daily digest

4. **Monitoring Metrics:**
   - Items classified (total)
   - Distribution by category (%)
   - Distribution by priority (%)
   - P0/P1 escalations (count)
   - Escalation rate (P0+P1 / total)

**Acceptance:**
- Triager script created and integrated
- First week of triage data collected in `.squad/digests/triage/triage-{month}.jsonl`
- P0/P1 escalations tested and working

## Operational Documentation

### QMD Extraction Workflow

**When:** Every Sunday 22:00 UTC (configurable)

**Process:**
1. GitHub Actions triggers `qmd-weekly.yml`
2. Collect all daily digests from past 7 days
3. Run `qmd-extract.ps1` with folder path
4. Apply 5-category KEEP/DROP rules
5. Generate `.squad/digests/archive/{week}.qmd.md`
6. Auto-commit if new items detected
7. Update metrics in `.squad/monitoring/qmd-health.md`

**Manual Override:**
```powershell
.\.squad\scripts\qmd-extract.ps1 -StartDate "2026-03-01" -EndDate "2026-03-08"
```

---

### Dream Routine Workflow

**When:** Every Monday 08:00 UTC (configurable)

**Process:**
1. GitHub Actions triggers `dream-routine.yml`
2. Load last 4 QMD digests (configurable, default N=4)
3. Run `dream-routine.ps1` with digest list
4. Execute 6 analysis tasks (LLM-powered)
5. Generate `.squad/digests/dream/dream-{date}.md`
6. Auto-commit and notify leads
7. Flag skill promotion candidates for human review
8. Update metrics in `.squad/monitoring/dream-health.md`

**Manual Override:**
```powershell
.\.squad\scripts\dream-routine.ps1 -DigestCount 4 -OutputPath ".squad/digests/dream/"
```

---

### Issue-Triager Workflow

**When:** After each channel scan (daily)

**Process:**
1. Channel-scan completes, produces per-channel files
2. Digest-processor invokes `issue-triager.ps1` before merging
3. Classify each item by category + priority
4. Apply escalation rules (P0 → incident channel notification)
5. Write classified items to `.squad/digests/triage/triage-{month}.jsonl`
6. Merge into daily digest with priority labels
7. Update metrics in `.squad/monitoring/triager-health.md`

**Manual Review:**
```powershell
# View all P0/P1 items from today
Get-Content ".squad/digests/triage/triage-$(Get-Date -Format 'yyyy-MM').jsonl" |
  ConvertFrom-Json |
  Where-Object { $_.priority -in @('P0', 'P1') } |
  Format-Table -Property date, category, priority, original_item
```

## Risk Mitigation

| Risk | Mitigation |
|------|-----------|
| LLM classification errors | Include human review loop; track false positive rate; adjust thresholds |
| Memory tier not separated | Monthly `.gitignore` verification; CI rule to prevent raw files in commits |
| QMD extraction drops important signal | Quality checklist before finalization; Dream Routine feedback loop |
| Dream Routine insufficient data | Require minimum 2 QMD digests; mark confidence "Low" if <3 |
| P0 escalation noise | Require 3-dimension priority scoring, not keyword-only; tune thresholds |
| Workflow failures (GitHub Actions) | Log all errors to `.squad/monitoring/{component}-errors.jsonl`; alert on 3+ consecutive failures |
