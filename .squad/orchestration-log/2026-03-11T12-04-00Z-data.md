## Architecture

### Layer 1: Data Collection
- Polls `~/.squad/ralph-heartbeat.json` (2s interval)
- Scans `.squad/orchestration-log/*.md` (5s interval)
- Extracts agent spawns, board updates, completion status

### Layer 2: Event Processing
- `activityParser.ts` — Parse orchestration logs
- `useActivityPoller()` hook — Polling and state management
- `LiveActivityContext` — React context for component tree

### Layer 3: UI Rendering
- Live dashboard component showing active agents
- New keybinds: 'a' (agent view), 'l' (raw log)
- Real-time duration and status tracking

## Design Rationale

- **File-based:** Reliable, no dependency on noisy agency logs
- **Polling:** Simple state management, stateless design
- **Incremental reads:** `FileStream.Seek` avoids full re-parse
- **Active session heuristic:** Modified time + no shutdown event
