# טקסים — Ceremonies 🎉

> פגישות צוות שקורות לפני או אחרי עבודה. כל squad מגדיר את שלו.

## רטרוספקטיבה שבועית — מה למדנו השבוע? 🤔

| שדה | ערך |
|-----|------|
| **טריגר** | אוטומטי |
| **מתי** | אחרי — כל יום ראשון |
| **תנאי** | סוף שבוע עבודה |
| **מנחה** | אבא/אמא או ה-Squad Lead |
| **משתתפים** | כל הצוות |
| **זמן** | 15-20 דקות |
| **פעיל** | ✅ כן |

**סדר יום:**
1. 🌟 **מה הלך טוב?** — הצלחות השבוע
2. 🔧 **מה היה קשה?** — אתגרים שנתקלנו בהם
3. 💡 **מה למדנו?** — דברים חדשים שגילינו
4. 🎯 **מה רוצים לעשות בשבוע הבא?** — יעדים

**שאלות עזר:**
- "איזה issue היה הכי כיף לעבוד עליו?"
- "מה היה הדבר הכי מפתיע שלמדתם?"
- "אם הייתם יכולים לשנות משהו, מה זה היה?"

## סטנדאפ יומי — מה עשינו היום? 📋

| שדה | ערך |
|-----|------|
| **טריגר** | ידני (אופציונלי) |
| **מתי** | אחרי — סוף יום עבודה |
| **תנאי** | כשעובדים על פרויקט |
| **מנחה** | כל אחד מהצוות |
| **משתתפים** | מי שעבד היום |
| **זמן** | 5 דקות |
| **פעיל** | ✅ כן |

**כל אחד עונה על 3 שאלות:**
1. ✅ **מה עשיתי?** — "עבדתי על issue #5, הוספתי כפתור"
2. 📝 **מה אני הולך לעשות?** — "מחר אני רוצה לסיים את ה-PR"
3. 🚧 **האם יש משהו שתוקע אותי?** — "אני לא מבין איך לעשות X"

**טיפ:** אפשר לכתוב את הסטנדאפ כתגובה ב-issue במקום לדבר!

## סקירת קוד — Code Review 👀

| שדה | ערך |
|-----|------|
| **טריגר** | אוטומטי |
| **מתי** | אחרי PR |
| **תנאי** | כש-PR חדש נפתח |
| **מנחה** | @copilot או חבר צוות |
| **משתתפים** | מי שפתח את ה-PR + reviewer |
| **זמן** | 10-15 דקות |
| **פעיל** | ✅ כן |

**סדר יום:**
1. 📖 **קריאת הקוד** — מה השתנה?
2. 🧪 **בדיקה** — האם זה עובד?
3. 💬 **הערות** — מה אפשר לשפר?
4. ✅ **אישור** — Approve או בקשת שינויים

**כללי Code Review לילדים:**
- 👍 תמיד להתחיל במשהו חיובי
- 🎓 להסביר *למה* ולא רק *מה* צריך לשנות
- 🤝 לזכור — כולנו לומדים!
- 🎯 להתמקד בדברים חשובים, לא בסגנון
