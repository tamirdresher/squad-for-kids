# Copilot Notebooks Skill

## What Are GitHub Copilot Notebooks?

**GitHub Copilot Notebooks** (2026) are AI-powered, persistent workspaces that extend Copilot beyond code autocomplete. They serve as a home for understanding, synthesizing, and collaborating on complex projects involving diverse sources (documents, slides, PDFs, notes) and multiple contributors.

Unlike transient chat windows, Notebooks provide:
- **Persistent Context**: Long-term memory of all referenced materials
- **Rich Reference Management**: Add and update documents (Word, Excel, PowerPoint, PDFs) as project sources; edits auto-reflect
- **AI-Generated Overviews**: Intelligent dashboards summarizing key insights and themes; auto-refresh as data is added
- **Three-Column Layout**: References, Copilot-generated pages, and chat sidebar in one view—minimal context-switching
- **Audio & Video Summaries**: Quick catch-ups with customizable narration style and focus
- **Collaboration**: Real-time co-editing, action item extraction, playbook/charter generation
- **Enterprise Security**: Data protection and compliance for sensitive work

## Core Capabilities

| Feature | Benefit |
|---------|---------|
| Persistent AI Chat | Context preserved across sessions with all referenced materials |
| Overview Page | Auto-generated dashboard of themes, insights, and project status |
| Multi-Document Reference | Link Word, Excel, PowerPoint, PDFs—curate sources for Copilot to synthesize |
| Smart Instructions | AI-suggested prompts for planning, learning, and structured problem-solving |
| Co-Authoring | Share notebooks, collaborate in real-time, auto-generate documentation |
| Audio Recaps | Listen to project summaries instead of reading—great for async updates |

## When to Use Copilot Notebooks vs Regular Copilot

### **Use Regular Copilot (IDE, CLI) for:**
- Code generation and completion
- Quick, single-purpose Q&A
- Real-time pair programming in your editor
- Bug fixes, refactoring, test generation
- API/syntax lookups

### **Use Copilot Notebooks for:**
- **Research & Learning**: Collecting and synthesizing multiple sources over time
- **Project Planning**: Long-term, multi-contributor project tracking with AI insights
- **Documentation**: Drafting reports, charters, playbooks from scattered sources
- **Team Knowledge Work**: Making sense of complex, evolving projects with changing contributors
- **Meeting/Audit Prep**: Gathering materials and generating summaries for stakeholder reviews
- **Cross-Functional Collaboration**: Bringing together work from many disciplines (design, eng, product, legal)

## Integration with Squad Workflows

### **When Squad Members Would Use Notebooks**

1. **@picard (Lead) — Architecture & Planning**
   - Create a Notebook for major architectural decisions
   - Reference PRs, design docs, decision logs, RFCs
   - Have Copilot generate synthesis of pros/cons, impact analysis
   - Share with team for async feedback before finalizing

2. **@seven (Research & Docs) — Research & Knowledge**
   - Persistent research Notebook for learning new frameworks/patterns
   - Curate tutorials, blogs, official docs, code examples
   - AI-generate summaries and learning paths
   - Share insights with team

3. **Squad Planning & Retrospectives**
   - Create a Notebook for sprint planning
   - Reference backlog, prior sprint results, customer feedback, metrics
   - Have Copilot suggest priorities, identify risks
   - Generate sprint charter automatically

4. **Complex Issues/Bugs**
   - Notebook linking PR, issue, logs, related commits
   - AI synthesizes context for faster onboarding
   - Generates troubleshooting playbook

### **How to Integrate into Workflows**

- **Squad Decisions**: Create a Notebook for major decisions, link relevant context (ADO tickets, PRs, research), let Copilot synthesize options
- **Onboarding New Members**: Pre-built Notebook with key docs, code examples, architecture notes—new member reads or listens to audio summary
- **Post-Mortems**: Gather timeline, logs, related PRs in a Notebook; Copilot generates structured incident report
- **Quarterly Reviews**: Reference OKRs, project summaries, metrics; generate executive summary

## Technical Workflow

1. **Create Notebook** in Microsoft 365 Copilot or web interface
2. **Add References**: Drag-drop or link Word, Excel, PowerPoint, PDFs, URLs
3. **Chat with Copilot**: Ask questions, request summaries, synthesis
4. **Generate Artifacts**: Have Copilot create drafts, playbooks, reports
5. **Share & Collaborate**: Invite squad members, co-edit in real-time
6. **Auto-Overview**: Copilot maintains a living dashboard of project state

## Limitations & Considerations

- **Best for synthesis work**, not real-time coding
- **Requires M365 Copilot** (or Copilot Pro)
- **Not ideal for sensitive security data** without additional review of enterprise policies
- **Largest value with evolving, multi-source projects**—less useful for simple Q&A
- **Learning curve**: Team may need training on effective reference organization
