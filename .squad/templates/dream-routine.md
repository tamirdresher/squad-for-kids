# Dream Routine — Cross-Digest Analysis & Trend Detection

> Prompt template for nightly/weekly cross-referencing of digests.
> Detects trending topics, recurring patterns, and flags items for skill promotion.
> Source: OpenCLAW production pattern (https://trilogyai.substack.com/p/openclaw-in-the-real-world)
> Adopted for Squad: Issue #23

## Purpose

Named after human sleep — where memory consolidation and pattern recognition happen — the Dream Routine runs during off-peak hours and performs cognitive maintenance across accumulated digests. It bridges the gap between individual digest capture (Phase 1) and automated skill promotion (Phase 3).

## When to Run

- **Frequency:** Weekly (after QMD digest is generated), or on-demand
- **Input:** Last N QMD digests (default: N=4, i.e., last 4 weeks)
- **Output:** Dream Report (`.squad/digests/dream/dream-YYYY-MM-DD.md`)

---

## Dream Routine Prompt Template

Use this prompt when invoking the Dream Routine analysis. Replace `{variables}` with actual values.

```
You are performing a Dream Routine — a cross-digest analysis of the last {N} weeks
of QMD digests for the Squad team.

INPUT DIGESTS:
{Paste or reference the last N QMD digest files}

ANALYSIS TASKS:

1. TRENDING TOPICS
   Scan all {N} digests and identify topics/themes that appear in 3+ digests.
   For each trend, report:
   - Topic name
   - Frequency (appeared in X of {N} digests)
   - Trajectory (increasing, stable, decreasing)
   - First seen: {date}
   - Most recent: {date}
   - Summary of evolution across digests

2. RECURRING BLOCKERS
   Identify blockers that appear in 2+ digests without resolution.
   For each recurring blocker:
   - Description
   - First reported: {date}
   - Times reported: {count}
   - Current status (still blocked / partially resolved / workaround in place)
   - Recommended escalation action

3. INCIDENT PATTERN DETECTION
   Look for related incidents that may indicate a systemic issue.
   Cluster incidents by:
   - Root cause similarity
   - Affected component/service
   - Time correlation
   Flag clusters of 2+ related incidents as potential systemic issues.

4. DECISION DRIFT
   Compare decisions across digests. Flag:
   - Contradictory decisions (decided X in week 1, decided not-X in week 3)
   - Decisions that were made but never implemented
   - Decisions that keep being revisited without resolution

5. SKILL PROMOTION CANDIDATES
   Items that qualify for promotion to permanent skills (.squad/skills/):
   - Pattern appeared in 3+ digests (durability test)
   - Pattern has clear "if X then Y" structure (actionability test)
   - Pattern would help agents handle future similar situations (utility test)
   
   For each candidate, draft:
   - Skill name (kebab-case)
   - One-line description
   - Evidence (which digests, what examples)
   - Confidence: High (5+ appearances) / Medium (3-4) / Low (2, flag only)
   - Proposed SKILL.md outline

6. COMMITMENT TRACKING
   Cross-reference commitments across digests:
   - Commitments approaching deadline (next 7 days)
   - Commitments past deadline with no resolution update
   - Commitments met (celebrate / close)

OUTPUT FORMAT:
Use the Dream Report format specified below.
```

## Configuration

| Parameter | Default | Description |
|-----------|---------|-------------|
| `N` (digest window) | 4 | Number of QMD digests to analyze |
| `promotion_threshold` | 3 | Minimum appearances before flagging for skill promotion |
| `confidence_high` | 5 | Appearances needed for "High" confidence |
| `blocker_escalation_days` | 14 | Days a blocker persists before auto-escalation flag |
| `commitment_warning_days` | 7 | Days before deadline to flag "at risk" |

## Integration with Squad

```
Raw Digests → QMD Extraction → Dream Routine → Skill Promotion
   (daily)      (weekly)         (weekly)        (as needed)
```

- **QMD** feeds Dream Routine (Dream consumes QMD digests, not raw logs)
- **Dream Report** feeds Picard (lead reviews recommendations, approves promotions)
- **Skill Promotion** candidates go to `.squad/skills/` after human review
- **Commitment tracking** feeds into sprint planning / team ceremonies

## Guardrails

1. **Minimum data requirement:** Do not run Dream Routine with fewer than 2 QMD digests. Results below 3 digests should be marked "Low confidence — insufficient data."
2. **False positive mitigation:** Require 3+ data points for any trend claim. Single occurrences are noise.
3. **Human review required:** No skill promotion without human approval. Dream Routine recommends; humans decide.
4. **Scope limitation:** Dream Routine analyzes patterns, not causation. Flag "possible systemic issue" — don't diagnose root cause without investigation.
