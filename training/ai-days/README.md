# AI Days Training Artifacts

This directory captures learning artifacts, session notes, patterns, and Copilot SDK knowledge from AI Days sessions.

## Directory Structure

- **`sessions/`** — Session notes organized by date. Use `TEMPLATE.md` to structure new sessions.
- **`patterns/`** — Reusable patterns discovered during AI Days. Document solutions, approaches, and anti-patterns here.
- **`copilot-sdk/`** — Copilot SDK patterns and code examples. See `README.md` in that directory for guidance.

## How to Add Session Notes

1. Create a new markdown file in `sessions/` with the naming convention: `YYYY-MM-DD-title.md`
   - Example: `2026-03-15-squad-coordination.md`
2. Use the `TEMPLATE.md` as a starting point.
3. Include:
   - Session date and attendees
   - Key learnings and takeaways
   - Code snippets or commands demoed
   - Links to relevant repos or PRs
   - Action items and follow-ups

## How to Add Patterns

1. Create a new markdown file in `patterns/` describing a reusable pattern.
2. Use the `TEMPLATE.md` as a starting point.
3. Include:
   - Problem statement
   - Solution overview
   - When to use and when to avoid
   - Code examples
   - Links to implementations in this repo

Refer to `copilot-sdk/README.md` for guidance on documenting Copilot SDK patterns and examples.
