EXECUTIVE SUMMARY
─────────────────

Issue #999 proposes mapping GitHub issue `needs:*` labels to Kubernetes node labels
for pod scheduling. Example: `needs:gpu` → `nodeSelector: {nvidia.com/gpu: "true"}`.

STATUS: ✅ Design Complete & Ready for Implementation
EFFORT: 4–5 weeks (6 phased implementation phases)

KEY DECISIONS RATIFIED
──────────────────────

1. Node Labels (not Pod Labels)
   └─ Capabilities are infrastructure properties, not workload declarations

2. Hybrid Approach: Static Node Pool + Dynamic DaemonSet
   ├─ GPU, memory tier → node pool labels (fast, AKS config)
   └─ Browser, secrets, mounts → DaemonSet discovery (responsive)

3. Hard Requirements via nodeSelector
   └─ `needs:*` labels → Pod stays pending if requirement unsatisfied
   └─ Future: `prefers:*` labels → affinity rules (optional, weighted)

4. Label Namespace: squad.io/capability-*
   └─ Consistent, collision-free, easy to audit

5. RBAC: Capability discoverer gets nodes/patch
   └─ Minimal necessary permission; Phase 2: add admission webhook
