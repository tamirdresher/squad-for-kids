זוכרים [איך Squad הפך למערכת הפרודוקטיביות הראשונה שבאמת עבדה בשבילי](/blog/2026/03/10/organized-by-ai)? לא בגלל שאני השתניתי — אלא בגלל שהמערכת לא צריכה שאני אזכור. Ralph עוקב אחרי התור. החלטות נרשמות. הצוות רץ בזמן שאני ישן.

זה היה פריצת הדרך האישית שלי בפרודוקטיביות. AI שלא שוכח. AI שלא צריך כוח רצון.

אבל הנה העניין: ברגע שיש לך שבעה סוכני AI שבאמת עובדים — Picard, Data, Worf, Seven, B'Elanna, Podcaster, Neelix — ועוד Ralph שמנטר הכל מסביב לשעון, אתה מפסיק לחשוב עליהם כסקריפטים לאוטומציה. אתה מתחיל לחשוב עליהם כ**צוות**.

וצוותים צריכים יותר מ-prompts. הם צריכים קליטה. מבנה. ידע. הקשר.

![Borg cube](/assets/scaling-ai-part1-first-team/borg-resistance-is-futile.png)
*"ההתנגדות חסרת תועלת. ה-backlog שלכם ייטמע."*

## מ-"היי AI, תתקן את זה" ל-"צוות, הנה התוכנית"

ב[פוסט של חלק 0](/blog/2026/03/10/organized-by-ai), הראיתי לכם את לולאת ה-5 דקות של Ralph — בדיקת issues, מיזוג PRs, תיעוד החלטות. זה עדיין רץ. עדיין עובד.

אבל הבנתי שהתייחסתי ל-Squad כמו רשימת מטלות משופרת, כשבעצם זה משהו הרבה יותר.

השינוי קרה כשהפסקתי לומר "תתקן את באג האימות" והתחלתי לומר "**צוות:** תתקנו את באג האימות."

המילה הזו — **צוות** — שינתה את הדרך שבה Squad מגיב. במקום ש-Data (מומחה הקוד שלי) קופץ ישר לתיקון, Picard (המוביל שלי) נכנס קודם:

```
🎖️ Picard: מנתח באג אימות...
   ניתוח: רענון טוקן JWT נכשל בסשנים שפג תוקפם
   תלויות: צריך להבין את לוגיקת פקיעת הטוקן לפני התיקון
   
   → Data: סקור את זרימת האימות, זהה את שורש הבעיה
   → Worf: בדוק השלכות אבטחה (טוקנים שפג תוקפם, חטיפת סשנים)
   → Seven: עדכן את תיעוד האימות עם הנמקת התיקון
   
   צפי: 3 זרמים מקבילים, Data מסיים ראשון, השאר עוקבים
```

Picard לא רק הקצה את המשימה ל-Data. הוא **ניתח** אותה, **זיהה תלויות**, ו**פיזר את העבודה למומחים הנכונים**. זהו פירוק משימות (task decomposition), וזה ההבדל בין "סוכן אחד עושה הכל" ל"צוות שמתאם".

זו לא טכנולוגיה חדשה — ככה [Brady Gaster](https://github.com/bradygaster) תכנן את Squad מלכתחילה. מה שחדש זה שאני סוף סוף משתמש בזה בדרך שלשמה זה נבנה. וברגע שעשיתי את זה, הקפיצה בפרודוקטיביות לא הייתה הדרגתית. היא הייתה אקספוננציאלית.

## ה-`squad init` הראשון שלך (או: איך קראתי שמות לצוות)

כיסיתי את היסודות של `squad init` ב[חלק 0](/blog/2026/03/10/organized-by-ai), אבל הנה החלק שחשוב לבניית צוות אמיתי: מערכת ההקצאה (casting). ל-Squad יש 31 יקומים בדיוניים מובנים, והבחירה היא דטרמיניסטית על בסיס שם ה-repo שלכם. אותו repo, אותו צוות בכל פעם.

עבור ה-repo האישי שלי, Squad בחר ב-**Star Trek: Voyager**:

| תפקיד | סוכן | התמחות |
|--------|-------|---------|
| 🎖️ מוביל | **Picard** | ארכיטקטורה, החלטות, האצלה (שיניתי את שמו מ-Janeway) |
| 🔧 Backend | **Data** | סקירת קוד, C#, Go, עומק .NET |
| 🔒 אבטחה | **Worf** | ניתוח אבטחה, compliance, שרשרת אספקה |
| 📋 מחקר | **Seven** | תיעוד, מחקר, סינתזה |
| ⚙️ תשתיות | **B'Elanna** | Kubernetes, ענן, CI/CD |

השמות הם לא קוסמטיים. כל סוכן מקבל **פרסונה** שמעצבת את אופן התקשורת שלו. Picard אסטרטגי והחלטי. Data יסודי ומדויק. Worf אגרסיבי לגבי מקרי קצה באבטחה. Seven ישירה ואנליטית. זה נשמע כמו גימיק עד שרואים אותם בפעולה — מערכת הפרסונות בפועל מייצרת גישות שונות באופן משמעותי לאותה בעיה.

## קליטת ה-Squad שלך (החלק שכולם מדלגים עליו)

הנה מה שרוב האנשים מפספסים: **squad init זה לא הסוף. זו ההתחלה.**

כשהקמתי את הצוות לראשונה, לא סתם קיבלתי את ההרכב ברירת המחדל והתחלתי להקצות issues. בפועל קלטתי אותם. כמו מהנדסים אמיתיים שמצטרפים לצוות אמיתי.

הדבר הראשון שעשיתי היה לומר ל-squad לסרוק הכל. כל ה-repo. לא רק קוד — מוסכמות, תיעוד, החלטות ארכיטקטוניות, תבניות. הפניתי אותם לוויקי הפנימי שלנו, נתתי להם קישורים לתיעוד רלוונטי, שיתפתי רשומות החלטות ארכיטקטוניות מצוותים אחרים. כי squad בלי הקשר זה פשוט grep יקר. הייתי צריך שהם **יבינו** את ה-codebase, לא רק יריצו פקודות עליו.

אז הם סרקו. הם אינדקסו. הם בנו מאגר ידע משלהם. וצפיתי בהם לומדים. Data זיהה את דפוסי הטיפול בשגיאות שלנו. Seven קטלגה את מבנה התיעוד שלנו. Worf מיפה את בקרות האבטחה שלנו. B'Elanna למדה את pipeline ההפצה שלנו.

זה השלב שבו ה-squad שלכם עובר מ"עוזרי AI גנריים" ל"חברי צוות שבאמת מכירים את הפרויקט שלכם." והנה ההבדל המכריע מקליטת מהנדס אנושי: הם לא רק צורכים את הידע הזה באופן פסיבי. הם **מתחזקים** אותו. כל פעם שמתקבלת החלטה, היא נכנסת ל-`.squad/decisions.md`. כל פעם שמתגלה דפוס, הוא הופך ל-skill. הידע מצטבר עם הזמן.

קליטה מסורתית היא אירוע חד-פעמי. קליטת Squad היא **מתמשכת**. הם תמיד לומדים, תמיד מתעדכנים, תמיד נעשים חכמים יותר לגבי ה-codebase הספציפי שלכם.

## נתינת משימה אמיתית ראשונה

כאן זה מתחבר. אתם נותנים ל-Squad שלכם משימה:

```
Team, build the user search feature with filtering and pagination
```

Picard לוקח פיקוד. בתור מוביל, הוא לא מתחיל לכתוב קוד — הוא **מתזמר**:

```
🎖️ Picard: מפרק משימת חיפוש משתמשים...
   → Data: בנה API חיפוש עם תמיכה בסינון
   → Seven: הוסף תיעוד API עם דוגמאות סינון
   → Worf: בדוק סניטציה של קלט (סיכון SQL injection)
   → B'Elanna: הוסף הגדרות pagination לפריסת ה-API
```

כל ארבעת הסוכנים מתחילים לעבוד **בו-זמנית**. Data כותב את endpoint החיפוש בזמן ש-Seven מנסחת את התיעוד בזמן ש-Worf בודק פגיעויות injection בזמן ש-B'Elanna מעדכנת את הגדרת ההפצה. זה לא סדרתי — זה מקבילי באמת.

הפעם הראשונה שראיתי את זה קורה, פשוט ישבתי וצפיתי בטרמינל גולל. ארבעה סוכנים, ארבעה ענפי עבודה, כולם מתקדמים בו-זמנית. המטאפורה של הטמעת ה-Borg אינה מקרית — זה באמת מרגיש כמו תודעה קולקטיבית שצוללת על ה-codebase שלכם.

וזכרו: [לא הייתי צריך לנסח prompt בשביל זה](/blog/2026/03/10/organized-by-ai). הלולאה של Ralph כבר ראתה את ה-GitHub issue עם תגית `squad:picard`, הקצתה אותו לצוות, והתחילה את העבודה. פשוט התעוררתי עם ארבעה PRs בסקירה.

## Ralph — עדיין בלתי נלאה

כיסיתי את Ralph ב[פוסט של חלק 0](/blog/2026/03/10/organized-by-ai), אבל אני רוצה להדגיש: **Ralph הוא מה שגורם למערכת לעבוד בלעדיכם.**

Ralph פועל בשלוש שכבות, וכל אחת מרחיבה את ההישג של צוות ה-AI שלכם. הלולאה בתוך הסשן היא סריקה-טריאג'-הקצאה בסיסית בתוך סשן Copilot הנוכחי שלכם. אחר כך `squad watch` רץ כ-daemon מקומי שנשאר בחיים גם כשמפעילים מחדש את הטרמינל — כך שהלולאה ממשיכה גם כשסוגרים טאב. אבל משנה המשחק האמיתי הוא ה-heartbeat של GitHub Actions: workflow מתוזמן שמריץ את Ralph ב-CI, ללא השגחה כלל. צוות ה-AI שלכם עובד בזמן שאתם ישנים.

התעוררתי עם PRs ממוזגים שאף פעם לא נגעתי בהם. Ralph ראה את ה-issue, הקצה אותו ל-Data, Data תיקן, הבדיקות עברו, Ralph מיזג. אפס התערבות אנושית. לזה התכוונתי ב[חלק 0](/blog/2026/03/10/organized-by-ai) כשאמרתי "AI לא שוכח, AI לא צריך כוח רצון." Ralph רץ כל 5 דקות, לנצח, בין אם אתם ליד המחשב ובין אם לא. מערכות פרודוקטיביות נכשלות כשהן דורשות זכירה. Ralph לא דורש זכירה.

## החלטות וזיכרון (ידע ארגוני ששורד)

הראיתי לכם את `.squad/decisions.md` ב[פוסט של חלק 0](/blog/2026/03/10/organized-by-ai) — הקובץ היחיד שבו כל החלטה משמעותית נתפסת עם הנמקה מלאה, לא רק מסקנות.

הנה מה שלא הבנתי לגמרי בהתחלה: **decisions.md הוא לא רק לוג. הוא המוח המשותף של הצוות.**

כל פעם שסוכן מתחיל משימה, הוא קורא את decisions.md קודם. כשהוא מקבל החלטה משמעותית, הוא כותב אותה בחזרה. זה אומר שהצוות שלכם צובר **ידע ארגוני** לאורך סשנים. בסשן 1, Data מחליט להשתמש ב-bcrypt לגיבוב סיסמאות ומתעד את זה ב-decisions.md. עד סשן 5, Seven כותבת תיעוד אימות ומתייחסת להחלטת ה-bcrypt אוטומטית — כי היא כתובה שם ב-decisions.md. עד סשן 10, Worf מבקר פיצ'ר איפוס סיסמה ומאשר תאימות bcrypt בלי שמישהו מזכיר לו. ההחלטה הפיצה את עצמה.

לכל סוכן יש גם **history.md** — יומן הלמידה האישי שלו. ה-history של Data עוקב אחרי כל API שבנה, כל החלטת מסד נתונים, כל אופטימיזציית ביצועים. עם הזמן, סוכנים מפתחים **מומחיות אמיתית ב-codebase הספציפי שלכם**.

ואז יש **skills** — תבניות לשימוש חוזר שסוכנים מגלים ומשתפים. כש-Data מבין את מוסכמת הטיפול בשגיאות של הפרויקט שלכם, הוא לוכד אותה כ-skill. בפעם הבאה ש-Seven צריכה לטפל בשגיאות בדוגמאות תיעוד, ה-skill הזה זמין לה. ידע לא רק נשמר — הוא **זורם לאורך הצוות**.

לזה התכוונתי כשאמרתי ש-Squad הפך ל[מערכת הפרודוקטיביות הראשונה שלא נטשתי](/blog/2026/03/10/organized-by-ai). מערכות מסורתיות מסתמכות על כך שאתם תתחזקו אותן. Squad מתחזק **את עצמו**. הידע מצטבר.

## הגדלת הצוות (כשברירות המחדל לא מספיקות)

זוכרים כשאמרתי ש-Squad בוחר את הצוות שלכם מתוך יקומים בדיוניים? זה ההרכב ההתחלתי שלכם. אבל **אתם לא תקועים איתו**.

במהלך שלב הקליטה שלי, הבנתי שלצוות הבסיסי חסר משהו. הפרויקט שלי כלל עבודת תשתיות Azure — רשתות, ניהול משאבים, אינטגרציה עם שירותים. לאף אחד מהסוכנים ההתחלתיים שלי לא הייתה מומחיות עמוקה ב-Azure. אז הוספתי מומחה Azure להרכב. אחר כך צצה עבודת רגולציה — בקרות אבטחה, מעקב ביקורת, דרישות פנימיות שצריך לעקוב אחריהן. אף אחד בצוות הבסיסי לא היה בעלים של התחום הזה, אז הוספתי גם מומחה compliance.

ככה צוותים אמיתיים גדלים. לא מגייסים את כולם ביום הראשון. מעריכים את העבודה, מזהים פערים, ומביאים את המומחיות שנדרשת. Squad עובד באותו אופן. ההרכב ההתחלתי נותן לכם בסיס, אבל ככל שהפרויקט שלכם מתפתח, ה-squad שלכם יכול להתפתח גם. מומחי תחום, מומחי כלים, סוכנים ייעודיים לתפקידים — אתם מעצבים את הצוות לפי העבודה.

ובדיוק כמו עם מהנדסים אמיתיים, אתם לא סתם מוסיפים אותם ומצפים שיהיו פרודוקטיביים מיד. אתם קולטים אותם. נותנים להם הקשר. מראים להם את המוסכמות. נותנים להם לסרוק את ה-repo ולבנות את המודל המנטלי שלהם. ההבדל הוא שעם סוכני AI, קליטה לוקחת דקות במקום שבועות.

## חברי Squad אנושיים (הפיצ'ר שמשנה הכל)

כאן הסיפור מקבל תפנית. כל מה שהראיתי לכם עכשיו — Picard מאציל משימות, לולאת הניטור של Ralph, סוכנים עובדים במקביל — כל זה ב-**repo האישי שלי**. רק אני וצוות ה-AI שלי.

אבל הנה השאלה שלא הפסקתי לחשוב עליה: **מה קורה כשאתה לא האדם היחיד?**

כי אני לא רק עובד על repos אישיים. יש לי עבודה. ב-Microsoft. בצוות פלטפורמת תשתיות. עם מהנדסים אחרים שיש להם מומחיות אמיתית, דעות אמיתיות, וסמכות merge אמיתית. אי אפשר סתם לזרוק צוות AI לתוך זה ולומר "הטמיעו את ה-backlog." חברי הצוות שלכם לא נרשמו לסוכני AI שמקבלים החלטות בשלוש בלילה.

ואז קראתי את התיעוד של Brady על **חברי squad אנושיים**, והכל התחבר.

אפשר להוסיף בני אדם להרכב ה-Squad. אנשים אמיתיים, עם handles אמיתיים ב-GitHub, שמוקצים לתפקידים אמיתיים. כשעבודה מנותבת לחבר צוות אנושי, Squad לא ממציא את התשובה שלו ולא מדלג על השלב — הוא **עוצר וממתין**.

הוספתי את עצמי להרכב כחבר squad אנושי:

```markdown
## Human Members

- **Tamir Dresher** (@tamirdresher) — Human Squad Member  
  - Role: AI Integration Lead
  - Expertise: AI workflows, DevOps automation, C#/.NET
  - Scope: Squad adoption, agent orchestration, integration patterns
```

עכשיו כש-Picard צריך את הקלט שלי בסקירה ארכיטקטונית, Squad עוצר ושולח לי הודעה. צוות ה-AI ממשיך לעבוד על כל מה שלא תלוי בסקירה שלי. כשאני מגיב, Squad תופס את החוט וממשיך.

**זה הפיצ'ר שהופך את Squad למוכן לסביבה ארגונית.** זה אומר ש-Squad לא מחליף את הצוות שלכם — הוא **מגביר** אותו. מהנדסים בכירים עדיין בעלים של החלטות קריטיות. סקירות אבטחה עדיין עוברות דרך בני אדם. אבל עבודת המימוש, פיגומי הבדיקות, סנכרון התיעוד — כל אלה מטופלים על ידי חברי squad שהם AI בזמן שחברי squad אנושיים מתמקדים בארכיטקטורה ושיקול דעת.

וכאן מתחיל הפרק הבא. מה אם הייתי מוסיף את **חברי הצוות האמיתיים שלי** כחברי squad אנושיים? מה אם [Brady Gaster](https://github.com/bradygaster) — הבחור שיצר את Squad — הופך לחבר squad בצוות העבודה שלי? מה אם מנהל האבטחה היה סוקר את ממצאי Worf לפני שהם יוצאים? מה אם כללי ניתוב מגדירים מתי AI עוצר ומסלים לבני אדם?

מה אם צוות העבודה עצמו הופך ל-Squad — בני אדם ו-AI עובדים יחד, לא במקביל?

זה [חלק 2](/blog/2026/03/12/scaling-ai-part2-collective). ושם הדברים נהיים באמת מעניינים.

## פיצ'רים למשתמש מתקדם (מה שהתיעוד לא מדגיש מספיק)

[הפוסטים של Brady על Squad](https://github.com/bradygaster/squad) מכסים את הארכיטקטורה בצורה מצוינת. אבל הנה הפיצ'רים שגיליתי כמשתמש יומיומי ושינו את תהליך העבודה שלי בדרכים שלא ציפיתי.

**Export/Import** מאפשר לכם לארוז את הידע המצטבר של הצוות — decisions.md, skills, כללי ניתוב — לחבילה ניידת עם `squad export`, ואז להזריק אותו ל-repo חדש עם `squad import`. במקום לחזור על שלושה שבועות של למידה, ה-Squad של ה-repo החדש שלי היה פרודוקטיבי מהסשן הראשון. העברתי את הצוות בין repos בלי לאבד ידע ארגוני.

**`squad doctor`** מאמת את כל ההגדרה שלכם — 9 בדיקות שמכסות קבצי קונפיגורציה, הגדרות סוכנים, חיבורי upstream, ועוד. הכל ירוק אומר שאתם בסדר. כל בעיה מקבלת דיאגנוסטיקה ברורה עם הצעה לתיקון. אני מריץ את זה אחרי כל שינוי קונפיגורציה. זה חסך לי מסשני "למה זה לא עובד?" יותר פעמים ממה שאני יכול לספור.

**Notifications** שינו את האינטראקציה היומיומית שלי עם Squad. Squad שולח לי הודעה ב-Teams כשהוא צריך קלט. אני לא צריך לעקוב אחרי הטרמינל. כש-Picard צריך את חתימתי על סקירה, או כש-Worf מוצא בעיית אבטחה שדורשת שיקול דעת אנושי, אני מקבל נוטיפיקציה בטלפון. אני מגיב ב-Teams, ו-Squad תופס את החוט. זה מה שהופך את "human in the loop" ל**מעשי באמת** — אתם לא מרותקים לשולחן וצופים בפלט סוכנים נגלל.

**OpenTelemetry + Aspire** נותן לי observability מלא למה שכל סוכן עושה. Traces, logs, metrics — אני יכול לראות כמה זמן משימות לוקחות ואיפה נוצרים צווארי בקבוק. כש-Data בילה 8 דקות על מה שהיה צריך להיות endpoint API של 2 דקות, יכולתי לראות בדיוק לאן הזמן הלך ב-trace waterfall. זה לא רק דיבוג — זה הבנה של מאפייני הביצועים של צוות ה-AI שלכם לאורך זמן.

**Context Optimization** פותר את הבעיה שנתקלתי בה אחרי שלושה שבועות: decisions.md הלך ותפח, ומספר הטוקנים עלה. Squad מגזם אותו אוטומטית — מאחד החלטות חוזרות, מעביר לארכיון ישנות, שומר על הקשר פעיל רזה. צפיתי בו יורד מ-80K טוקנים ל-33K בלי לאבד הקשר חשוב. הסוכנים שלי הפכו מהירים יותר כי הם לא שחו בהחלטות מיושנות על פיצ'רים שנשלחו לפני שבועיים.

**Remote Control** באמצעות `squad start --tunnel` חושף את הסשן שלכם דרך URL של devtunnel. פתחו אותו בטלפון ואתם שולטים בצוות ה-AI שלכם מהספה. בניתי את האינטגרציה הזו ו[כתבתי עליה כאן](/blog/2026/02/26/squad-remote-control). זה הפך לדרך ברירת המחדל שלי לנטר את Squad — להתחיל עבודה ליד המחשב, לבדוק התקדמות מהטלפון בהפסקת צהריים.

## מה הלאה: ממגרש משחקים אישי לייצור

הפוסט הזה כיסה repo בודד עם צוות Squad בודד — **מגרש המשחקים האישי שלי** שבו Picard יכול לקבל החלטות ארכיטקטוניות בשתיים בלילה ואף אחד לא מתלונן.

אבל אני לא רק עובד על repos אישיים. אני עובד על מערכות ייצור ב-Microsoft, עם חברי צוות אמיתיים שיש להם מומחיות, דעות, וסמכות merge. הצוות שלי מנהל תשתיות שעליהן שירותים אמיתיים תלויים. תקני סקירת קוד. סריקות אבטחה. שערי הפצה. אי אפשר פשוט לעשות copy-paste של הגדרת ה-Squad האישית לעולם הזה ולצפות שזה יעבוד.

השאלה שהשאירה אותי ער בלילה: **האם Squad יכול לעבוד בצוות הנדסה אמיתי?**

התשובה הסתברה ככן. אבל פריצת הדרך לא הייתה ללמד את Squad לעבוד מהר יותר. זה היה ללמד את Squad לעבוד **לצד** חברי הצוות האמיתיים שלי.

ב[חלק 2: "מ-Repo אישי לצוות עבודה"](/blog/2026/03/12/scaling-ai-part2-collective), אראה לכם מה קרה כשהוספתי את הקולגות שלי כ-**חברי squad אנושיים** להרכב. איפה [Brady Gaster](https://github.com/bradygaster) — הבחור שיצר את Squad — הפך בעצמו לחבר squad. איפה כללי ניתוב מגדירים מתי AI מטפל בעבודה שחורה ומתי בני אדם מטפלים בשיקולי דעת. איפה צוות העבודה עצמו הפך ל-Squad — בני אדם ו-AI, ביחד.

ההתנגדות חסרת תועלת. ה-backlog שלכם ייטמע. 🟩⬛
