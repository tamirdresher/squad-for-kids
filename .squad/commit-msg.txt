## Add a Podcaster/Broadcaster Agent to Squad

### Problem
Squad produces a LOT of text — research reports, blog drafts, patent analysis, architecture docs, daily briefings, issue summaries. Reading all of it is time-consuming. Tamir wants an agent that can convert these outputs into short audio briefings/podcasts he can listen to instead of read.

### Inspiration
- **Google NotebookLM** — generates conversational podcast-style audio summaries from documents
- **GitHub Copilot Desktop** — has text-to-speech/audio capabilities
- **Constraint:** We can only use Microsoft and GitHub tools (no Google services)

### What the Podcaster Would Do

1. **Research Digests** — When Seven finishes a research report, the Podcaster creates a 2-5 minute audio summary
2. **Daily Briefings** — Convert the daily RP status briefing (#200) into an audio version
3. **PR Summaries** — "Here's what your team shipped today" as a quick audio recap
4. **Blog Drafts** — Read back blog posts (#41) so Tamir can review by listening
5. **Decision Briefs** — When issues need Tamir's input (pending-user), create a short audio explaining the decision needed
6. **Sprint Recaps** — End-of-sprint audio summary of what was accomplished

### Implementation Research

#### Microsoft/GitHub Tools to Evaluate

1. **Azure AI Speech Service (Cognitive Services)**
   - Text-to-Speech with neural voices (very natural sounding)
   - SSML support for tone, pacing, emphasis
   - Multiple voices — could do "conversational" style with two voices
   - REST API callable from PowerShell/Node
   - Would need an Azure subscription + Speech resource

2. **Microsoft Edge TTS / PlayReady**
   - Edge has built-in read-aloud with natural voices
   - Could potentially use the edge-tts npm package (uses Edge's free TTS API)
   - No Azure subscription needed
   - Quality is very good for narration

3. **Azure OpenAI TTS**
   - OpenAI's text-to-speech models via Azure OpenAI Service
   - Very natural, supports multiple voices and styles
   - Available through Azure OpenAI endpoints

4. **GitHub Copilot Audio (if available)**
   - Copilot Desktop reportedly has audio capabilities
   - Research what's available and if it can be used programmatically

5. **PowerShell System.Speech**
   - Built-in Windows TTS (System.Speech.Synthesis)
   - Low quality but zero setup — good for MVP/prototype
   - `Add-Type -AssemblyName System.Speech; (New-Object System.Speech.Synthesis.SpeechSynthesizer).Speak("Hello")`

#### Architecture Options

**Option A: Post-processing Pipeline**
```
Agent completes research → Scribe logs it → Podcaster picks it up → Generates audio → Posts to Teams/saves to file
```

**Option B: On-demand**
```
Tamir asks "podcast the latest research" → Podcaster reads the doc → Generates audio → Plays/sends it
```

**Option C: Automated daily podcast**
```
Daily at 8:55 AM → Podcaster gathers overnight activity → Generates 3-min morning briefing audio → Sends via Teams
```

### Proposed Agent

```
🎙️  {CastName}  — Podcaster/Broadcaster    Audio summaries, briefings, content narration
```

- **Primary task:** Convert long-form text into short, digestible audio
- **Format:** MP3 or WAV files, 2-5 minutes typical
- **Delivery:** Teams webhook (audio attachment), local file, or direct playback
- **Style:** Professional but conversational, like a tech podcast host
- **Trigger:** Automatic after major research/reports, or on-demand
