## מה זה GitHub Actions?

**הסבר פשוט:** GitHub Actions הם עוזרים אוטומטיים שרצים על השרתים של GitHub. חשבו עליהם כרובוטים קטנים שעושים עבורכם משימות באופן אוטומטי — בלי שום התערבות אנושית.

לא צריך להתקין שום דבר. לא צריך להשאיר את המחשב דולק. GitHub מטפלים בהכל בענן.

**דוגמאות רלוונטיות לסקוואד לילדים:**
- 🕐 **תזכורות למידה יומיות** — אוטומציה שבודקת אם הילד נכנס היום
- 📊 **דוחות התקדמות** — יצירה אוטומטית של סיכומי למידה שבועיים
- 📋 **סיווג בקשות** — כשהילד פותח בקשת למידה חדשה, היא מסווגת אוטומטית
- 💓 **בדיקות בריאות** — כל 6 שעות, רובוט בודק שהכל עובד כמו שצריך

## מה חינם

### ריפו ציבורי (כמו סקוואד לילדים!)

| משאב | חינם | הערות |
|------|------|-------|
| **דקות GitHub Actions** | **ללא הגבלה** | ריפו ציבורי מקבל דקות חינם ללא הגבלה! |
| **אחסון לקבצים** | 500 מגה | לוגים, דוחות, תוצרים |
| **ריצות מקבילות** | 20 | כמה אוטומציות יכולות לרוץ במקביל |

> ✅ **סקוואד לילדים הוא ריפו ציבורי.** כל האוטומציות של GitHub Actions הן **חינם לגמרי וללא הגבלה על דקות**.

### ריפו פרטי (אם תהפכו את הFork לפרטי)

| משאב | חינם (GitHub Free) | הערות |
|------|---------------------|-------|
| **דקות GitHub Actions** | **2,000 דקות/חודש** | ~33 שעות אוטומציה |
| **אחסון** | 500 מגה | כמו ציבורי |

> 💡 גם אם תהפכו את הFork לפרטי, 2,000 דקות חינם בחודש זה יותר ממספיק לסקוואד לילדים. תצטרכו להריץ אוטומציות 33+ שעות רצופות כדי לחרוג מזה.

## מה עולה כסף

אנחנו מאמינים ב**שקיפות מלאה**. הנה מה ש*יכול* לעלות כסף:

### GitHub Copilot (המוח של הAI)

| תכנית | מחיר | מה מקבלים |
|--------|-------|-----------|
| **Copilot Free** | **$0/חודש (₪0)** | 2,000 השלמות קוד + 50 הודעות צ'אט בחודש |
| **Copilot Pro** | **$10/חודש (~₪37)** או $100/שנה (~₪370) | השלמות וצ'אט ללא הגבלה |
| **תלמיד/מורה** | **$0/חודש (₪0)** | Copilot Pro מלא — חינם דרך [GitHub Education](https://education.github.com) |
| **מתחזק קוד פתוח** | **$0/חודש (₪0)** | Copilot חינם למתחזקי פרויקטים מתאימים |

### GitHub Actions (מעבר לחינם)

רלוונטי רק אם הריפו **פרטי** וחורגים מ-2,000 דקות/חודש:

| סוג שרת | עלות לדקה | עלות לשעה |
|----------|-----------|-----------|
| Linux | $0.008 (~₪0.03) | $0.48 (~₪1.78) |
| Windows | $0.016 (~₪0.06) | $0.96 (~₪3.55) |
| macOS | $0.08 (~₪0.30) | $4.80 (~₪17.76) |

> 🎯 **חשוב:** סקוואד לילדים משתמש רק בשרתי Linux, והריפו ציבורי. **העלות שלכם: ₪0.**

### GitHub Codespaces (סביבת הפיתוח בענן)

| משאב | חינם | הערות |
|------|------|-------|
| שעות מעבד | **120 שעות-ליבה/חודש** | ~60 שעות על מכונה דו-ליבתית |
| אחסון | 15 ג'יגה/חודש | לקבצי הCodespace |

> 💡 120 שעות-ליבה = בערך 60 שעות למידה בחודש. זה בערך שעתיים ביום — מספיק לרוב הילדים!

## האוטומציות בפרויקט הזה

סקוואד לילדים כולל 4 אוטומציות. **כולן חינם בריפו ציבורי.**

### 1. 💓 `squad-heartbeat.yml` — בדיקת בריאות

| | |
|---|---|
| **מה עושה** | בודק issues שלא סווגו, מוסיף להם תגיות, ומזכיר על PRs ישנים |
| **מתי רצה** | כל 6 שעות (אוטומטי) + הפעלה ידנית |
| **טריגר** | `cron: '0 */6 * * *'` |
| **עלות** | **חינם** (ריפו ציבורי) |
| **זמן ריצה טיפוסי** | פחות מדקה |

**בעברית פשוטה:** כל 6 שעות, ראלף (הרובוט הידידותי שלנו) בודק אם יש בקשות למידה חדשות שלא סווגו. אם PR ישן חוכה מעל שבוע, ראלף משאיר תזכורת עדינה.

### 2. 📋 `squad-triage.yml` — סיווג אוטומטי של בקשות

| | |
|---|---|
| **מה עושה** | קורא issues חדשים, מסווג אותם (באג, תכונה, שיעורי בית, משחק, שאלה), ומפרסם תגובה ידידותית |
| **מתי רצה** | כשissue מקבל את התגית `squad` |
| **טריגר** | `issues: [labeled]` |
| **עלות** | **חינם** (ריפו ציבורי) |
| **זמן ריצה טיפוסי** | פחות מ-30 שניות |

**בעברית פשוטה:** כשהילד יוצר בקשה חדשה (כמו "אני רוצה לבנות משחק!"), המערכת מבינה אוטומטית מה הסוג ומוסיפה תגיות מתאימות. היא גם מפרסמת תגובה מעודדת בעברית.

### 3. 📊 `squad-board-sync.yml` — סנכרון לוח פרויקט

| | |
|---|---|
| **מה עושה** | שומר על לוח הפרויקט מסונכרן — מוסיף issues חדשים, מעביר פריטים שהושלמו ל"סיום" |
| **מתי רצה** | כשissues נפתחים/נסגרים + כל יום ראשון ב-08:00 (שעון ישראל) |
| **טריגר** | `issues: [opened, closed, reopened, labeled]` + `cron: '0 5 * * 0'` |
| **עלות** | **חינם** (ריפו ציבורי) |
| **זמן ריצה טיפוסי** | פחות מ-30 שניות |

**בעברית פשוטה:** לוח הפרויקט (בו רואים על מה עובדים) נשאר מעודכן אוטומטית. כשהילד מסיים פרויקט, הכרטיס עובר ל"סיום". כל יום ראשון בבוקר רץ סנכרון מלא.

### 4. 🏷️ `squad-issue-assign.yml` — שיוך עבודה

| | |
|---|---|
| **מה עושה** | מקצה עבודה לחברי הצוות (כולל Copilot) כשמוסיפים תגית `squad:` |
| **מתי רצה** | כשissue מקבל תגית `squad:*` |
| **טריגר** | `issues: [labeled]` |
| **עלות** | **חינם** (ריפו ציבורי) |
| **זמן ריצה טיפוסי** | פחות מ-30 שניות |

**בעברית פשוטה:** כשמסמנים issue עם `squad:copilot`, המערכת מקצה אותו אוטומטית ל-GitHub Copilot (סוכן הקידוד של AI), שייצור ענף ויתחיל לעבוד עליו. הילד יכול אז לסקור את הקוד כמו מפתח אמיתי!

## איך להגדיר משימות מתוזמנות

### תחביר Cron בהסבר פשוט

GitHub Actions משתמש ב"ביטויי cron" כדי להגדיר לוחות זמנים. ככה קוראים אותם:

```
┌───────── דקה (0-59)
│ ┌─────── שעה (0-23, שעון UTC)
│ │ ┌───── יום בחודש (1-31)
│ │ │ ┌─── חודש (1-12)
│ │ │ │ ┌─ יום בשבוע (0=ראשון, 6=שבת)
│ │ │ │ │
* * * * *
```

### לוחות זמנים נפוצים

| לוח זמנים | ביטוי Cron | בעברית פשוטה |
|-----------|------------|-------------|
| כל 6 שעות | `0 */6 * * *` | בדקה 0, כל 6 שעות |
| כל יום ב-07:00 (ישראל) | `0 4 * * *` | 04:00 UTC = 07:00 ישראל (UTC+3) |
| כל בוקר ימי חול | `0 4 * * 1-5` | שני עד שישי ב-07:00 ישראל |
| כל יום ראשון | `0 5 * * 0` | כל יום ראשון ב-08:00 ישראל |
| ב-1 לכל חודש | `0 5 1 * *` | יום ראשון לכל חודש ב-08:00 ישראל |
| כל 15 דקות | `*/15 * * * *` | 4 פעמים בשעה (לא מומלץ — בזבוז דקות) |

> ⚠️ **הערה על אזורי זמן:** GitHub Actions משתמש בשעון UTC. ישראל היא UTC+3 (חורף) או UTC+2 (קיץ). אז "07:00 ישראל" = "04:00 UTC" בחורף, "05:00 UTC" בקיץ.

### איך להוסיף אוטומציה משלכם

1. בFork שלכם, נווטו ל-`.github/workflows/`
2. צרו קובץ חדש (למשל `daily-check-in.yml`)
3. השתמשו בתבנית הזו:

```yaml
name: תזכורת למידה יומית

on:
  schedule:
    # כל יום ב-07:00 שעון ישראל (04:00 UTC)
    - cron: '0 4 * * *'
  workflow_dispatch:  # מאפשר הפעלה ידנית

jobs:
  remind:
    runs-on: ubuntu-latest
    steps:
      - name: שליחת תזכורת
        run: echo "⏰ הגיע הזמן ללמוד! פתחו את הCodespace ואמרו שלום לצוות!"
```

4. עשו commit לקובץ לענף `main`
5. האוטומציה תתחיל לרוץ בזמן המתוזמן הבא

### איך לכבות אוטומציה שלא רוצים

1. היכנסו לFork שלכם ב-GitHub
2. לחצו על לשונית **"Actions"**
3. לחצו על שם האוטומציה בסרגל הצד
4. לחצו על כפתור **"..."** (שלוש נקודות) בפינה הימנית העליונה
5. בחרו **"Disable workflow"**

> 💡 כיבוי עדיף על מחיקה — אפשר להפעיל מחדש בכל עת.

## עלויות GitHub Copilot — המוח של הסקוואד

GitHub Copilot הוא מנוע הAI שגורם לסוכני הסקוואד לעבוד. הנה האפשרויות:

### Copilot Free — ₪0/חודש
- ✅ 2,000 השלמות קוד בחודש
- ✅ 50 הודעות צ'אט בחודש
- ✅ גישה למודלים GPT-4o ו-Claude Sonnet
- ⚠️ 50 הודעות צ'אט = מספיק לכ-2-3 פגישות למידה בשבוע

### Copilot Pro — ~₪37/חודש ($10) או ~₪370/שנה ($100)
- ✅ השלמות קוד ללא הגבלה
- ✅ הודעות צ'אט ללא הגבלה
- ✅ גישה לכל מודלי הAI
- ✅ מומלץ ללומדים יומיים

### Copilot לתלמידים ומורים — ₪0/חודש 🎓
- ✅ **הכל ב-Copilot Pro — חינם לגמרי**
- ✅ הרשמו ב-[education.github.com](https://education.github.com)
- ✅ דורש אימייל בית ספרי או הוכחת לימודים
- ✅ בתוקף למשך תקופת הלימודים

> 🏫 **אם הילד שלכם תלמיד (גם בבית ספר יסודי), ייתכן שהוא זכאי לתכנית החינוך החינמית.** תהליך ההרשמה פשוט ובדרך כלל מאושר תוך ימים ספורים.

## שאלות נפוצות

### "אקבל חיובים מפתיעים?"
לא. חשבונות GitHub Free מגיעים עם מגבלת הוצאות של $0 כברירת מחדל. אי אפשר לחרוג מהחינם אלא אם תוסיפו אמצעי תשלום ותעלו את מגבלת ההוצאות באופן מפורש.

### "מה קורה כשנגמרות ההודעות החינמיות של Copilot?"
הצ'אט פשוט מפסיק להגיב עד החודש הבא. בלי חיובים, בלי הפרעה לשום דבר אחר.

### "אפשר לשדרג/לשנמך את Copilot בכל עת?"
כן. אפשר לעבור בין Free ל-Pro בכל זמן מ-[github.com/settings/copilot](https://github.com/settings/copilot). אין חוזה או התחייבות.

### "המידע של הילד שלי משמש לאימון AI?"
GitHub Copilot לא משתמש בקוד או בשיחות שלכם לאימון מודלים. ראו את [הצהרת הפרטיות של GitHub](https://docs.github.com/en/copilot/overview-of-github-copilot/about-github-copilot-individual#about-privacy).
