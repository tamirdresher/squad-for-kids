# Memory Tier Separation — .gitignore Rules Reference

> Operational guide for enforcing the three-tier memory architecture.
> Part of Issue #66: OpenCLAW Adoption

---

## Overview

Squad's memory is separated into three tiers, each with different retention policies, access patterns, and git visibility. This document defines the `.gitignore` rules that enforce this separation.

**Reference:** `.squad/templates/memory-separation.md`

---

## Tier 1: Transaction (Raw) — GITIGNORED ❌

**Purpose:** Temporary operational data. High volume, time-bound relevance.

**Directory Structure:**

```
.squad/digests/
├── {date}.md                    # Daily unified digest (raw/merged)
├── {date}-{channel}.md          # Per-channel raw scans
└── triage/
    └── triage-{month}.jsonl     # Raw incident classification logs

.squad/log/
├── orchestration-log/           # Orchestration event logs
└── run-output/                  # Script execution output

.squad/sessions/
└── *.md                         # Session transcripts
```

**Rationale for Ignoring:**
- Ephemeral: Replaced weekly by QMD extracts
- High volume: Would bloat repository
- May contain PII: Email addresses, user names, project details
- Intermediate state: No long-term value

**`.gitignore` Rules:**

```gitignore
# Tier 1 — Transaction Layer (Raw, Temporary)
.squad/digests/*.md
.squad/digests/*-*.md
.squad/digests/triage/*.jsonl
.squad/log/**/*
.squad/sessions/*.md
.squad/sessions/**/*.md
```

**Verification:**
```powershell
# These should be IGNORED (no output from check-ignore means ignored)
git check-ignore ".squad/digests/2026-03-11.md"
git check-ignore ".squad/digests/2026-03-11-dk8s-support.md"
git check-ignore ".squad/digests/triage/triage-2026-03.jsonl"
git check-ignore ".squad/log/orchestration-log/2026-03-11.md"
git check-ignore ".squad/sessions/session-2026-03-11.md"
```

## Tier 2: Operational (Curated) — COMMITTED ✅

**Purpose:** Processed, curated organizational memory. Permanent reference value.

**Directory Structure:**

```
.squad/digests/archive/
├── 2026-03-01-to-07.qmd.md      # Weekly QMD digest (curated)
├── 2026-03-08-to-14.qmd.md
└── {start}-to-{end}.qmd.md

.squad/digests/dream/
├── dream-2026-03-08.md          # Monday cross-digest analysis
├── dream-2026-03-15.md
└── dream-{date}.md

.squad/decisions/
├── inbox/                       # Pending decision records
│   └── {agent}-{issue}.md
└── merged/                      # Finalized decisions (part of history)
    └── decision-{id}.md
```

**Rationale for Committing:**
- Permanent value: Reference material for months/years
- Curated: Processed through KEEP/DROP filters
- Searchable: Git log, blame, history recovery
- Critical context: Decision history, trend patterns

**`.gitignore` Rules:**

```gitignore
# Tier 2 — Operational Layer (Curated, Committed)
# ✅ EXPLICITLY NOT IGNORED — these files MUST be committed

# Note: No entries needed. By default, Git commits all files not explicitly ignored.
# The absence of ignores in this section is the rule.
```

**Verification:**
```powershell
# These should NOT be IGNORED (check-ignore returns nothing)
git check-ignore ".squad/digests/archive/2026-03-01-to-07.qmd.md"
# Should output: .squad/digests/archive/2026-03-01-to-07.qmd.md

git check-ignore ".squad/digests/dream/dream-2026-03-08.md"
# Should output: .squad/digests/dream/dream-2026-03-08.md

git check-ignore ".squad/decisions/inbox/seven-openclaw-adoption.md"
# Should output: .squad/decisions/inbox/seven-openclaw-adoption.md
```

## Enforcement Rules

### Rule 1: Automatic Ignoring (Git)

File a `.gitignore` at `.squad/` level:

```gitignore
# Transaction Layer (Tier 1) — Ignored by default
digests/*.md
digests/*-*.md
digests/triage/*.jsonl
log/**/*
sessions/*.md
sessions/**/*.md
```

### Rule 2: Positive Assertion (CI/CD)

In CI pipeline, verify no Tier 1 files are staged:

```powershell
# Fail if any Tier 1 files are in git staging area
$stagedTier1 = git diff --cached --name-only | Where-Object {
    $_ -match "\.squad/digests/\d{4}-\d{2}-\d{2}\.md" -or
    $_ -match "\.squad/digests/triage/" -or
    $_ -match "\.squad/log/" -or
    $_ -match "\.squad/sessions/"
}

if ($stagedTier1) {
    Write-Error "CI BLOCKING: Raw Tier 1 files must not be committed: $stagedTier1"
    exit 1
}
```

### Rule 3: Monthly Audit

First Monday of each month, verify separation:

```powershell
# Run monthly audit
function Audit-MemorySeparation {
    # Tier 1: Should be gitignored
    $tier1Files = @(
        ".squad/digests/2026-03-11.md",
        ".squad/digests/2026-03-11-dk8s-support.md",
        ".squad/digests/triage/triage-2026-03.jsonl"
    )
    
    # Tier 2/3: Should NOT be gitignored
    $tier23Files = @(
        ".squad/digests/archive/2026-03-01-to-07.qmd.md",
        ".squad/digests/dream/dream-2026-03-08.md",
        ".squad/skills/squad-conventions/SKILL.md"
    )
    
    Write-Host "Verifying Tier 1 files are gitignored..."
    foreach ($file in $tier1Files) {
        $ignored = git check-ignore $file
        if (-not $ignored) {
            Write-Warning "NOT IGNORED (should be): $file"
        }
    }
    
    Write-Host "Verifying Tier 2/3 files are NOT gitignored..."
    foreach ($file in $tier23Files) {
        $ignored = git check-ignore $file
        if ($ignored) {
            Write-Warning "IGNORED (should not be): $file"
        }
    }
    
    Write-Host "Audit complete."
}

Audit-MemorySeparation
```

## Special Cases

### Case 1: Retaining Tier 1 Data (Archival)

If raw digests must be retained beyond Tier 1 retention window (30 days), manually promote to Tier 2:

1. Copy raw digest to `.squad/digests/archive/` with QMD metadata
2. Re-commit as archive entry
3. Original raw file can be deleted from filesystem (still gitignored, not tracked)

### Case 2: Incident Post-Mortems

Incident analysis documents are Tier 2 (curated), not Tier 1 (raw):

```
.squad/digests/postmortems/
├── 2026-03-05-outage-eastus2.md     # ✅ COMMITTED (analysis, not raw logs)
└── 2026-03-10-cert-expiry-incident.md
```

Raw incident logs remain Tier 1 (gitignored); only the processed post-mortem is committed.

### Case 3: Sensitive Data in Tier 1

If Tier 1 files contain sensitive data (customer IDs, PII, secrets):
- Tier 1 is gitignored → safe from git history
- Before promoting to Tier 2, sanitize all sensitive references
- Use [anonymization] placeholders: "[CUSTOMER-ID]", "[USER-EMAIL]"

## Integration with Workflows

### QMD Extraction Workflow

```
Tier 1: .squad/digests/{date}.md (raw dailies)
   ↓ (weekly, Sunday 22:00 UTC)
Tier 2: .squad/digests/archive/{week}.qmd.md (curated weekly)
   ↓ (committed automatically)
Git history (searchable, permanent)
```

### Dream Routine Workflow

```
Tier 2: .squad/digests/archive/{week}.qmd.md (input)
   ↓ (weekly, Monday 08:00 UTC)
Tier 2: .squad/digests/dream/dream-{date}.md (analysis output)
   ↓ (committed automatically)
Git history (searchable, permanent)
```

### Issue-Triager Workflow

```
Tier 1: .squad/digests/triage/triage-{month}.jsonl (raw classifications)
   ↓ (processed daily)
Tier 2: .squad/digests/archive/{week}.qmd.md (signaled items retained in QMD)
   ↓ (committed weekly)
Git history (searchable, permanent)

Note: Raw triage JSONL is not committed (Tier 1, gitignored).
Only classified items that pass KEEP filter become Tier 2.
```

## Troubleshooting

### Issue: Tier 1 File Was Committed

**Symptom:** Raw digest file (.squad/digests/2026-03-11.md) appears in git log.

**Resolution:**
```powershell
# Remove from git history (destructive — requires force push)
git rm --cached ".squad/digests/2026-03-11.md"
git filter-branch --tree-filter 'rm -f .squad/digests/2026-03-11.md' HEAD~10..HEAD
git push --force-with-lease origin squad/66-openclaw-adoption

# Or: If recent commit, just amend
git reset HEAD .squad/digests/2026-03-11.md
git commit --amend --no-edit
git push --force-with-lease
```

### Issue: Tier 2 File Is Gitignored

**Symptom:** `.squad/digests/archive/2026-03-01-to-07.qmd.md` doesn't show in git add.

**Resolution:**
```powershell
# Check why file is ignored
git check-ignore -v ".squad/digests/archive/2026-03-01-to-07.qmd.md"

# Remove from .gitignore if overly broad rule exists
# Verify correct .gitignore rules are in place (see "Enforcement Rules" above)

# Force add (bypassing gitignore)
git add --force ".squad/digests/archive/2026-03-01-to-07.qmd.md"
git commit -m "Force add: Tier 2 QMD digest (should not have been ignored)"
```
