Core Architecture Patterns
🔄 Continuous Watch Loop
Agent polls issue queue on a fixed interval. No human trigger needed.
while(true) { scan(); act(); sleep(5min); }
🌳 Task Decomposition
Lead agent breaks complex tasks into parallel sub-tasks for specialists.
task → analyze → fan-out → parallel → merge
📋 Routing Table
Work-type → agent mapping. Clear ownership prevents conflicts.
security? → Worf | code? → Data | docs? → Seven
🧠 Shared Memory (Git-Native)
decisions.md captures every choice. Agents read past decisions before acting.
.squad/decisions.md → version-controlled brain
📦 Drop-Box Pattern
Agents leave artifacts for others in known locations. Async coordination.
Data → writes code → Seven reads → writes docs
🔁 Auto-Recovery
Failed tasks re-enter the queue. Ralph retries with context from failure.
fail → log reason → re-queue → retry with context

Anti-Patterns — What NOT to Do
⚠️
One-agent-does-everything: A single general-purpose agent can't maintain context across security, code, docs, and infrastructure. Specialize.
⚠️
No shared memory: Without decisions.md, agents repeat mistakes and contradict each other. Git-native state is non-negotiable.
⚠️
Manual triggering only: If you must remember to start agents, you've built a better CLI — not a team. Add Ralph.
✅
Do this instead: Specialized agents + routing table + shared memory + continuous monitoring = autonomous team.
