**By:** Tamir Dresher (via Copilot)
**What:** Squad for Kids should gradually teach kids AND parents to use GitHub tools as part of the learning experience:
- **Issues** — the kid learns to create issues for homework tasks ("I need help with fractions"), the squad picks them up
- **Project Board** — visual kanban of what the kid is learning, what's done, what's next. Parents can see it.
- **PRs** — when the kid completes a project, the squad opens a "PR" that the parent reviews (like grading but fun)
- This creates **digital literacy as a side effect** — the kid learns GitHub workflow without knowing it's a developer tool
- The squad should introduce these concepts gradually, using kid-friendly language ("your mission board", "submit your work for review", "your learning backlog")
**Why:** Teaching kids GitHub skills while they learn math/science is a massive secondary benefit. By the time they're teenagers they already know how developers work. Plus it gives parents a familiar dashboard (the project board) to track progress.
