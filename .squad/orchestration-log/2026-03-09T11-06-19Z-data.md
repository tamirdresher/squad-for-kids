## Task Assignment

Design architecture for a "live agent activity panel" that provides real-time visibility into active agent sessions during Ralph's orchestration rounds.

Requirements:
- Display which sub-agents are running
- Show what tools they're calling
- Indicate session health status
- Integrate with existing squad-monitor dashboard

**Post to:** tamirdresher_microsoft/tamresearch1#207

## Work Completed

1. **Analysis Phase:**
   - Examined ralph-watch.ps1 orchestration logs (event structure, timing)
   - Reviewed squad-monitor codebase (Program.cs, dashboard architecture)
   - Analyzed data sources: heartbeat.json, orchestration logs, agency session logs, process list
   - Identified Tamir's implementation notes on what works/doesn't

2. **Architecture Design:**
   - **Three-layer model:** Data Collection → Event Processing → UI Presentation
   - **Data source priority:** Heartbeat JSON > Orchestration Logs > Process List
   - **Polling strategy:** 2s heartbeat checks, 5s orchestration-log scans, zero agency log tailing
   - **Active session heuristic:** events.jsonl modified within 2 min AND no shutdown event

3. **Implementation Plan:**
   - Single-file architecture preserved (~300 LOC added to Program.cs, stays under 1800 total)
   - Incremental file reading with FileStream.Seek (avoid full re-parse every 5s)
   - Agent correlation via toolCallId (subagent.started → subagent.completed)
   - Three view modes: 'a' (processed agent view), 'l' (raw log), summary bar on main dashboard

4. **Decision Documented:**
   - `.squad/decisions/inbox/data-207-monitor-design.md`: Live Agent Activity Panel Architecture
